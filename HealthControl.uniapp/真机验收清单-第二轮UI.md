# 真机验收清单（第二轮 UI 精修）

> 目标：验证「首页-自查-服务中心-我的」四导航信息架构、核心链路可用性、样式稳定性与旧跳转兼容性。  
> 端：微信小程序优先，H5 兼容性做基础核对。

## 1. 底部导航与结构

- [ ] 底部导航仅在 4 个 tab 页面可见：`Index / AudioRecord / ServiceHub / UserCenter`
- [ ] 底部导航高亮与当前页面一致
- [ ] 点击当前 tab 不重复跳转
- [ ] 4 个 tab 之间互跳无白屏、无回退异常
- [ ] iOS 刘海屏与安卓底部安全区无遮挡

## 2. 首页（主流程优先）

- [ ] 顶部主按钮“开始检测”可跳转到 `AudioRecord`
- [ ] “查看检测记录”可跳转到 `AudioRecordList`（未登录时拦截到登录）
- [ ] “健康档案”入口可跳转到 `HealthRecord`（未登录时拦截到登录）
- [ ] “服务中心”入口可跳转到 `ServiceHub`
- [ ] 扩展服务区入口均可达（科普/饮食/问卷/就医/AI/消息）
- [ ] 未读消息角标显示正确（0 / 1-99 / 99+）

## 3. 自查主链路

- [ ] `AudioRecord` 录音按钮状态切换正常（开始/录中/继续）
- [ ] 选择文件上传路径正常（mp3/wav/m4a/flac）
- [ ] 上传后可进入 `AudioResult`
- [ ] `AudioResult` 中“再次检测/历史记录/记录症状/AI解读/AI咨询/就医辅助/上传喉镜”按钮均可跳转
- [ ] 检测成功与失败态 UI 均正常展示

## 4. 服务中心聚合

- [ ] 健康管理分区入口可达：档案/症状/提醒/记录/喉镜
- [ ] 知识与工具分区入口可达：科普/问卷/饮食/AI/标注
- [ ] 就医与隐私分区入口可达：就医/隐私/导出/删除/协议/历史
- [ ] 需要登录的入口有拦截与跳转登录
- [ ] 不需登录的入口可直接浏览

## 5. 我的页与账户能力

- [ ] 用户卡片信息展示正常（头像/昵称/角色/微信绑定状态）
- [ ] 最近测评卡片点击可进入 `QuestionnaireHome`
- [ ] 常用服务、数据与隐私、账户设置三组入口均可达
- [ ] “退出登录”后 token 与用户状态清空，并跳回登录页

## 6. 旧交互兼容回归（高风险）

- [ ] `AudioRecordList` 长按多选/全选/置顶/删除逻辑正常
- [ ] `SymptomLogList` 筛选、详情跳转、新增跳转正常
- [ ] `HealthRecord` 各卡片入口（症状/检测/提醒/就医/标注/喉镜）正常
- [ ] 消息中心详情页 action 跳转仍正确
- [ ] 登录态丢失场景下重定向不出现循环跳转

## 7. 样式与可读性

- [ ] 页面背景、卡片、按钮、导航颜色保持统一视觉体系
- [ ] 主次按钮层级清晰（主按钮高对比，次按钮弱化）
- [ ] 小屏设备不出现卡片溢出、文字截断不可读
- [ ] 长列表页滚动时无明显卡顿与闪动

## 8. H5 基础兼容

- [ ] H5 首页、服务中心、我的页排版不崩
- [ ] H5 `AudioRecord` 录音按钮有“浏览器不支持录音”提示
- [ ] H5 文件上传路径可正常走检测流程

---

## 建议执行顺序

1. 先测四导航与首页主链路  
2. 再测自查链路（录音/上传 -> 结果 -> 跳转）  
3. 再测服务中心与我的页所有入口  
4. 最后做登录态、异常态和 H5 兼容回归
