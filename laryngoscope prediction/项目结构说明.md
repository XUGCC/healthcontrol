# 项目结构说明

## 1. 项目当前的主线

这个仓库本质上是一个“喉镜图像预测 + 热力图可视化”的桌面端原型，主流程分成两层：

1. `demo.py`
   负责整个桌面系统入口，包含界面、病例管理、数据库交互、喉镜/音频/视频三个子功能入口。
2. `larynscope/`
   负责喉镜图像相关的模型、预测、热力图和简单图像编辑。

就喉镜图像这条链路来说，当前仓库里实际存在两套相关模型：

1. `larynscope/net.pth`
   被 [demo.py](/c:/Users/徐国城/Desktop/喉镜图预测/demo.py) 通过 [pic.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/pic.py) 调用，用于图像分类预测。
2. `store/repalexnet_heatmap_n100.pth`
   被 [show_single_heatmap.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/show_single_heatmap.py) 调用，用于基于 `RepAlexNet` 生成 Grad-CAM 热力图。

这也是为什么你会看到“旧系统预测模型”和“热力图模型”在代码里不是同一个文件。

## 2. 根目录文件与目录作用

### [demo.py](/c:/Users/徐国城/Desktop/喉镜图预测/demo.py)

整个桌面系统的主程序，体量很大，主要职责是：

- 启动 PyQt 界面
- 管理登录、病例、图片列表、数据库记录
- 在喉镜图像子窗口里调用图像预测与热力图生成

和你当前任务最相关的调用链在这里：

- `pic.detect_soft_NBI(self.img)`：先做分类预测
- `show_single_heatmap.run(self.img, predict)`：再按预测类别生成热力图

也就是说，旧系统里“分类”和“热力图”是两个独立模块拼起来的。

### [larynscope](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope)

喉镜相关代码包。

#### [model_repAlexnet.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/model_repAlexnet.py)

`RepAlexNet` 网络定义文件，包含：

- `RepAlexBlock`
- `RepAlexNet`
- `repvgg_model_convert()`

其中 `repvgg_model_convert()` 会把训练态多分支结构转换成部署态结构。
[show_single_heatmap.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/show_single_heatmap.py) 就是先构造这个模型，再加载 `repalexnet_heatmap_n100.pth` 权重。

#### [utils_heatmap.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/utils_heatmap.py)

热力图核心工具，包含：

- `GradCAM`
- `show_cam_on_image()`

这是整个“中间产物热力图”真正的生成逻辑。

#### [show_single_heatmap.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/show_single_heatmap.py)

单张图热力图生成脚本，职责很明确：

- 加载 `RepAlexNet`
- 加载 `store/repalexnet_heatmap_n100.pth`
- 指定目标层 `model.stage5`
- 根据传入类别 `Class_Name` 生成一张 Grad-CAM 叠加图

它只负责“给定图片 + 给定类别，产出热力图”，不负责自己先做分类预测。

#### [pic.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/pic.py)

旧系统里用于图像分类预测的模块。它的特点是：

- 使用 `timm.create_model("vit_base_r50_s16_224.orig_in21k", ...)`
- 加载 [net.pth](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/net.pth)
- 输出 6 类中的一个类别编号

所以从代码关系上看，`pic.py` 对应的是“分类模型”，不是 `RepAlexNet`。

#### [model_hybird_cnnchannelAttention_serial_featureAttention.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/model_hybird_cnnchannelAttention_serial_featureAttention.py)

另一个模型定义文件，是 `ViT + 注意力模块` 的实验/训练模型代码。
它和 [pic.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/pic.py) 中用的 `timm` 模型路线有关，更像旧分类模型的来源或实验代码。

#### [bianji.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/bianji.py)

Tkinter 写的简单图像编辑工具，支持：

- 缩放
- 手动画线标注
- 保存编辑后的图片

它不参与模型预测，只是图像辅助编辑。

#### [__init__.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/__init__.py)

空包标记文件，用来让 `larynscope` 作为 Python 包被导入。

Python 运行后生成的缓存文件，对理解业务逻辑没有帮助，可以忽略。

## 3. 当前喉镜预测架构怎么串起来

旧系统里的喉镜图像流程是：

1. 在 [demo.py](/c:/Users/徐国城/Desktop/喉镜图预测/demo.py) 里选中图片
2. 调用 [pic.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/pic.py) 用 `net.pth` 做分类
3. 拿预测出来的类别编号，传给 [show_single_heatmap.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/show_single_heatmap.py)
4. 由 `RepAlexNet + GradCAM` 生成热力图
5. 把热力图保存到目录，再回显到界面上

## 4. 这次新增脚本采用的思路

你希望直接围绕 `repalexnet_heatmap_n100.pth` 来完成：

- 输入图片
- 进行预测
- 产出热力图
- 输出病变类型

因此我新增的脚本没有继续依赖旧 UI 和 `pic.py`，而是直接：

1. 用 [model_repAlexnet.py](/c:/Users/徐国城/Desktop/喉镜图预测/larynscope/model_repAlexnet.py) 构建模型
2. 加载 [store/repalexnet_heatmap_n100.pth](/c:/Users/徐国城/Desktop/喉镜图预测/store/repalexnet_heatmap_n100.pth)
3. 对输入图片做推理得到类别
4. 用同一个模型对预测类别生成 Grad-CAM 热力图
5. 把结果保存为 JSON + 热力图文件

这样更适合你现在要的“独立脚本批量跑图”场景。
