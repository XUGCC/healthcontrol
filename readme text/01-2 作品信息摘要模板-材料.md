# 作品简介与特别声明

## 作品简介（100字以内）

本作品面向居家喉部健康管理，融合小程序、Spring Boot 后端、本地音频模型、喉镜影像预测与 AI 辅助解读，提供自查、档案、提醒、饮食、就医和隐私数据管理闭环。


## 创新描述（100字以内）

项目创新在于把本地模型初筛、喉镜热力图、Qwen 多模态解读与健康档案打通；检测、症状、饮食、就医、隐私和标注授权形成可追踪闭环，AI 只做解释与建议，不替代诊断。


## 特别声明

1. 本作品不包含疆域地图；如后续加入地图导航、区域展示等内容，将使用自然资源部标准地图服务或具备有效审图号的合规地图，并在作品中标注来源与审图号。
2. 本作品的前期基础包括已有前后端框架、数据库设计、本地音频识别模型、喉镜影像 RepAlexNet 模型权重、测试样例及部分需求文档。本次主要工作为整合小程序端、管理端、Spring Boot 后端、Python 推理脚本与 AI 辅助解读能力，完善音频自查、喉镜预测热力图、健康档案、饮食建议、就医辅助、隐私数据权利和模型标注闭环等功能。
3. 本作品使用人工智能辅助工具。系统功能中接入阿里云百炼 DashScope 的通义千问 Qwen / Qwen-VL，用于音频图谱、喉镜图像的辅助解读、健康建议和问答支持；开发与材料整理过程中使用 ChatGPT / Codex 辅助需求梳理、代码排查和文案生成。AI 生成内容均经人工审核，且仅作为解释、建议或开发辅助，不承担核心检测结论，不替代医生诊断。AI 辅助内容约占整体作品的少量辅助部分，估算不超过 15%。
4. 项目中的检测结果、喉镜预测、热力图、AI 解读和健康建议仅供学习、演示、竞赛展示及居家健康参考，不构成医学诊断、治疗意见或处方建议；如出现持续声音嘶哑、吞咽痛、出血、呼吸困难等情况，应及时前往正规医疗机构就诊。
5. 项目涉及的音频、喉镜图片、健康档案、问卷、饮食、就医和隐私数据均应遵循最小必要、授权使用、脱敏展示和权限隔离原则。公开演示或提交材料时，不应包含真实用户身份信息、联系方式、病历隐私、未授权医学影像或真实密钥配置。


## 开发制作工具

- 前端与小程序：HBuilderX、微信开发者工具、uni-app、Vue 3、Pinia、uni-ui、SCSS。
- 管理后台：VS Code、Vue 3、Vite、Element Plus、Axios、Pinia、ECharts。
- 后端服务：IntelliJ IDEA、JDK 17、Spring Boot 3.3.1、Maven、MyBatis-Plus、MySQL、JWT、Apache POI。
- 模型与数据处理：Python、PyTorch、torchvision、librosa、matplotlib、OpenCV、Pillow、NumPy。
- AI 辅助与接口：阿里云百炼 DashScope、通义千问 Qwen / Qwen-VL、HTTP API、Base64 图像传输。
- 协作与测试：Git、PowerShell、Postman/接口调试工具、浏览器开发者工具。
