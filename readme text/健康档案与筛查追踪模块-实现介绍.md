## 二、主要实现功能

### 2.1 后端功能实现

#### 2.1.1 健康趋势计算服务（HealthTrendService）

**实现文件**：
- `HealthControl.springboot/src/main/java/com/example/web/service/HealthTrendService.java`
- `HealthControl.springboot/src/main/java/com/example/web/service/impl/HealthTrendServiceImpl.java`

**核心功能**：
1. **健康趋势计算**（`calculateHealthTrend`）
   - 基于最近3次检测记录，自动判定健康趋势
   - 返回结果：`好转`、`稳定`、`恶化`、`无数据`
   - 判定逻辑：
     - 异常→正常 = 好转
     - 正常→异常 = 恶化
     - 结果一致时，根据置信度变化判定趋势

2. **风险等级计算**（`calculateRiskLevel`）
   - 基于检测结果、症状严重程度、持续时间综合评分
   - 返回结果：`0`（低风险）、`1`（中风险）、`2`（高风险）
   - 评分规则：
     - 检测结果得分：0-60分（异常且置信度高时得分）
     - 症状严重程度得分：0-30分（重度30分，中度15分）
     - 持续时间得分：-5到10分（>1个月+10分，<1周-5分）

#### 2.1.2 健康档案服务扩展（TPersonalLaryngealHealthRecordServiceImpl）

**实现文件**：
- `HealthControl.springboot/src/main/java/com/example/web/service/impl/TPersonalLaryngealHealthRecordServiceImpl.java`

**新增方法**：
1. **GetByUserId**：根据用户ID获取健康档案
   - 用于前端健康档案页面展示
   - 自动关联最新检测记录信息

2. **AutoUpdateAfterDetect**：检测完成后自动更新档案
   - 更新末次自查时间和最新检测ID
   - 自动计算并更新健康趋势标签
   - 自动计算并更新风险等级

#### 2.1.3 症状日志服务扩展（TSymptomLogServiceImpl）

**实现文件**：
- `HealthControl.springboot/src/main/java/com/example/web/service/impl/TSymptomLogServiceImpl.java`

**增强功能**：
- 在 `CreateOrEdit` 方法中：新增或修改症状记录后，自动触发健康档案风险等级重新评定
- 在 `Delete` 方法中：删除症状记录后，自动触发健康档案风险等级重新评定

#### 2.1.4 音频检测控制器集成（AudioDetectController）

**实现文件**：
- `HealthControl.springboot/src/main/java/com/example/web/controller/AudioDetectController.java`

**集成功能**：
- 检测成功后，自动调用 `TPersonalLaryngealHealthRecordServiceImpl.AutoUpdateAfterDetect`
- 实现检测记录与健康档案的自动关联和更新

## 三、运行方法

### 3.1 环境要求

**后端（Spring Boot）**：
- JDK 17 或更高版本
- Maven 3.6+
- MySQL 数据库
- Python 3.x（用于音频检测）

**前端（Uni-app）**：
- Node.js 14+ 和 npm/yarn
- HBuilderX 或支持 Uni-app 的 IDE
- 微信开发者工具（如需要测试微信小程序）

### 3.2 后端启动步骤

1. **配置数据库**
   - 确保 MySQL 数据库已创建
   - 检查 `application.yml` 中的数据库连接配置
   - 确保 `TPersonalLaryngealHealthRecord` 和 `TSymptomLog` 表已创建

2. **启动 Spring Boot 应用**
   ```bash
   cd HealthControl.springboot
   mvn spring-boot:run
   ```
   或使用 IDE 直接运行主类

3. **验证后端启动**
   - 访问 `http://localhost:7245`（根据实际配置调整端口）
   - 检查控制台日志，确认无错误

### 3.3 前端启动步骤

1. **安装依赖**（如未安装）
   ```bash
   cd HealthControl.uniapp
   npm install
   ```

2. **配置 API 地址**
   - 检查 `utils/http.js` 中的 `baseURL` 配置
   - 确保指向正确的后端地址（如 `http://localhost:7245`）

3. **运行到 H5**
   - 在 HBuilderX 中打开项目
   - 选择"运行" → "运行到浏览器" → "Chrome"
   - 或使用命令行：
     ```bash
     npm run dev:h5
     ```

4. **运行到微信小程序**（可选）
   - 在 HBuilderX 中选择"运行" → "运行到小程序模拟器" → "微信开发者工具"
   - 或使用命令行：
     ```bash
     npm run dev:mp-weixin
     ```

### 3.4 登录准备

1. **注册/登录账号**
   - 访问首页，点击"注册"创建账号
   - 或使用已有账号登录

2. **获取 Token**
   - 登录成功后，系统会自动保存 Token
   - Token 用于后续 API 请求的身份验证

## 五、注意事项

### 5.1 数据库字段类型问题

**问题描述**：
- `TSymptomLog` 实体类中 `SymptomLevel` 字段当前定义为 `Boolean` 类型
- 根据需求文档，该字段应为 `Integer` 类型（1=轻度，2=中度，3=重度）

**影响**：
- 当前风险等级计算中，症状严重程度得分暂时跳过
- 前端表单中严重程度选择功能正常，但后端存储可能不符合预期

**解决方案**：
1. 检查数据库表结构，确认 `SymptomLevel` 字段的实际类型
2. 如果是 `INT` 类型，修改实体类和 DTO 类定义：
   - `TSymptomLog.java`：`private Boolean SymptomLevel` → `private Integer SymptomLevel`
   - `TSymptomLogDto.java`：`private Boolean SymptomLevel` → `private Integer SymptomLevel`
3. 修改后，启用 `HealthTrendServiceImpl.calculateRiskLevel` 中的症状严重程度得分计算逻辑

## 六、常见问题排查

### 6.1 健康档案不显示

**可能原因**：
- 用户未完成任何检测
- 检测状态未完成（`DetectTotalStatus` 为 `false`）
- 后端 API 调用失败

**排查步骤**：
1. 检查用户是否已完成至少一次检测
2. 检查检测记录的 `DetectTotalStatus` 字段
3. 查看浏览器控制台和网络请求，确认 API 调用是否成功

### 6.2 风险等级不更新

**可能原因**：
- 症状记录未保存成功
- 风险等级计算逻辑未触发
- 数据库字段类型不匹配

**排查步骤**：
1. 检查症状记录是否成功保存到数据库
2. 查看后端日志，确认 `calculateRiskLevel` 是否被调用
3. 检查 `SymptomLevel` 字段类型是否符合预期

### 6.3 健康趋势显示"无数据"

**可能原因**：
- 用户检测记录少于3条
- 检测记录状态未完成

**排查步骤**：
1. 确认用户已完成至少3次检测
2. 确认所有检测记录的 `DetectTotalStatus` 为 `true`
3. 查看后端日志，确认 `calculateHealthTrend` 的查询结果

### 6.4 症状记录无法关联检测记录

**可能原因**：
- 检测记录列表加载失败
- 检测记录 ID 传递错误

**排查步骤**：
1. 检查 `loadDetectRecords` 方法是否正常执行
2. 查看网络请求，确认检测记录列表 API 是否返回数据
3. 检查表单提交时的 `DetectId` 参数是否正确
