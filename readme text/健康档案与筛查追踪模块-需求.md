# 健康档案与筛查追踪模块 - 详细需求文档

## 一、模块概述

### 1.1 模块定位
健康档案与筛查追踪模块是居家喉部健康自查小程序的核心数据管理模块，旨在为用户建立专属的喉部健康档案，实现症状自主记录、检测历史追踪、健康趋势分析、风险等级评定等功能，形成"一人一档"的健康数据闭环。

### 1.2 核心价值
- **数据积累**：自动收集用户检测记录、症状日志，形成完整健康档案
- **趋势追踪**：通过历史数据对比，自动判定健康趋势（好转/稳定/恶化）
- **风险预警**：基于检测结果和症状，自动评定风险等级（低/中/高）
- **健康复盘**：提供可视化图表，帮助用户了解自身喉部健康变化

### 1.3 与其他模块的关系
- **依赖模块**：用户基础模块（用户信息）、核心音频自查分析模块（检测记录）
- **支撑模块**：为饮食推荐模块、就医辅助模块提供风险等级和症状数据
- **数据来源**：音频检测记录表（TAudioScreenRecord）、症状日志表（TSymptomLog）

## 二、功能需求详细说明

### 2.1 个人喉部健康档案管理

#### 2.1.1 档案自动创建与更新
**功能描述**：
- 用户首次完成音频检测后，系统自动创建个人健康档案
- 每次检测完成后，自动更新档案中的相关字段
- 支持手动编辑档案中的部分信息（症状、护嗓习惯等）

**业务规则**：
1. 创建时机：用户首次完成检测且检测状态为"完成"时自动创建
2. 更新时机：
   - 每次检测完成后，更新"末次自查时间"和"关联最新检测"
   - 根据最近3次检测结果，自动计算并更新"健康趋势标签"
   - 根据最新检测结果和症状，自动计算并更新"风险评估等级"
3. 编辑权限：用户可手动编辑"近期主要不适症状"、"症状持续时间"、"日常护嗓习惯"

#### 2.1.2 档案信息展示
**展示内容**：
1. **基础信息卡片**
   - 用户头像、昵称
   - 档案创建时间
   - 末次自查时间（格式：YYYY-MM-DD HH:mm）

2. **风险等级标识**
   - 低风险：绿色标签，显示"低风险"
   - 中风险：黄色标签，显示"中风险"
   - 高风险：红色标签，显示"高风险"
   - 未评定：灰色标签，显示"待评定"

3. **健康趋势标签**
   - 好转：绿色向上箭头 + "好转"
   - 稳定：灰色横线 + "稳定"
   - 恶化：红色向下箭头 + "恶化"
   - 无数据：灰色横线 + "暂无数据"

4. **近期症状摘要**
   - 显示最近一次记录的症状类型和严重程度
   - 如无症状记录，显示"暂无症状记录"

5. **护嗓习惯**
   - 显示用户填写的日常护嗓习惯（如：多喝水、避免大声说话等）

6. **最新检测结果卡片**
   - 检测时间
   - 检测结果（正常/异常）
   - 置信度
   - 点击可跳转到检测结果详情页

#### 2.1.3 检测历史趋势图
**功能描述**：
- 展示最近6次检测结果的趋势折线图
- X轴：检测时间（MM-DD格式）
- Y轴：置信度（0-100%）
- 不同结果用不同颜色区分：
  - 正常（良性）：绿色点
  - 异常（恶性）：红色点
  - 检测中：灰色点

**交互功能**：
- 点击图表上的点，可跳转到对应检测记录的详情页
- 支持左右滑动查看更多历史记录
- 无数据时显示"暂无检测记录"提示

### 2.2 症状日志自主记录

#### 2.2.1 症状记录表单
**必填字段**：
1. **症状类型**（单选）
   - 嘶哑
   - 疼痛
   - 吞咽困难
   - 咽干
   - 咽痒
   - 其他（可输入自定义症状）

2. **严重程度**（单选）
   - 轻度：轻微不适，不影响日常生活
   - 中度：明显不适，部分影响日常生活
   - 重度：严重不适，严重影响日常生活

3. **持续时长**（文本输入）
   - 格式：数字 + 单位（如：3天、2周、1个月）
   - 单位选项：天、周、月

**选填字段**：
1. **症状首次出现时间**（日期选择器）
   - 默认：当前时间
   - 可选择历史日期

2. **关联检测记录**（可选）
   - 下拉选择最近10次检测记录
   - 用于将症状与特定检测结果关联

3. **备注说明**（多行文本）
   - 最多200字
   - 可记录症状的详细描述、诱因等

#### 2.2.2 症状记录列表
**展示内容**：
- 症状类型标签（不同颜色区分）
- 严重程度标签（轻度/中度/重度）
- 持续时长
- 记录时间（MM-DD HH:mm格式）
- 关联检测记录（如有，显示检测结果和置信度）
- 操作按钮：查看详情、编辑、删除

**排序规则**：
- 默认按记录时间倒序（最新在前）
- 支持按症状类型、严重程度筛选

#### 2.2.3 症状记录详情页
**展示内容**：
- 症状完整信息（类型、程度、时长、出现时间、备注）
- 关联检测记录信息（如有）
- 操作按钮：编辑、删除

### 2.3 筛查历史与趋势追踪

#### 2.3.1 检测历史时间线
**展示方式**：
- 时间轴形式，按时间倒序展示
- 每个节点显示：
  - 检测时间
  - 检测结果（正常/异常）
  - 置信度
  - 症状记录（如有关联）

**交互功能**：
- 点击节点跳转到检测结果详情页
- 支持筛选：全部/正常/异常/检测中

#### 2.3.2 健康趋势自动判定
**判定规则**：
基于最近3次有效检测结果（状态为"完成"）进行趋势判定：

1. **好转**：
   - 最近一次为"正常"，且前一次为"异常"
   - 或最近一次置信度明显下降（下降>10%）

2. **稳定**：
   - 最近3次结果一致（均为正常或均为异常）
   - 或置信度变化在±5%以内

3. **恶化**：
   - 最近一次为"异常"，且前一次为"正常"
   - 或最近一次置信度明显上升（上升>10%）

4. **无数据**：
   - 有效检测记录少于3次

**更新时机**：
- 每次检测完成后自动计算
- 用户查看档案时实时计算

#### 2.3.3 声学参数变化对比
**功能描述**：
- 对比最近两次检测的声学参数（如Jitter、Shimmer、HNR、MPT）
- 以柱状图或折线图形式展示
- 标注参数变化趋势（上升/下降/持平）

**展示条件**：
- 仅当检测记录中包含声学参数数据时显示
- 无数据时显示"暂无声学参数数据"

---

### 2.4 风险等级自动评定

#### 2.4.1 评定规则
**评定依据**：
综合以下因素进行风险等级评定：

1. **检测结果**（权重：60%）
   - 异常（恶性）+ 高置信度（>70%）→ 高风险
   - 异常（恶性）+ 中置信度（50-70%）→ 中风险
   - 异常（恶性）+ 低置信度（<50%）→ 低风险
   - 正常（良性）→ 低风险

2. **症状严重程度**（权重：30%）
   - 重度症状 → 高风险
   - 中度症状 → 中风险
   - 轻度症状或无症状 → 低风险

3. **症状持续时间**（权重：10%）
   - 持续>1个月 → 风险等级+1级
   - 持续1周-1个月 → 风险等级不变
   - 持续<1周 → 风险等级-1级（最低为低风险）

**最终等级**：
- **低风险**：综合评分<40
- **中风险**：综合评分40-70
- **高风险**：综合评分>70

#### 2.4.2 评定时机
1. **自动评定**：
   - 每次检测完成后自动评定
   - 每次症状记录后自动重新评定
   - 症状记录删除后自动重新评定

2. **手动触发**：
   - 用户查看档案时，如数据有更新，自动重新评定

#### 2.4.3 风险提示
**高风险用户**：
- 档案页面顶部显示红色警告横幅
- 提示："您的风险等级为高风险，建议尽快就医"
- 提供"查看就医推荐"按钮

**中风险用户**：
- 档案页面显示黄色提示
- 提示："您的风险等级为中风险，建议定期自查并关注症状变化"

**低风险用户**：
- 显示绿色提示
- 提示："您的风险等级为低风险，请继续保持良好的护嗓习惯"

## 三、数据表设计

### 3.1 个人喉部健康档案表（TPersonalLaryngealHealthRecord）

**表结构**（基于现有表扩展）：

| 字段名 | 类型 | 说明 | 必填 | 备注 |
|--------|------|------|------|------|
| Id | Integer | 主键ID | 是 | 自增 |
| UserId | Integer | 关联用户ID | 是 | 外键，关联AppUser表 |
| RecentSymptom | String(500) | 近期主要不适症状 | 否 | 用户手动填写 |
| SymptomDuration | String(50) | 症状持续时间 | 否 | 如"1周"、"2个月" |
| DailyVoiceCare | String(500) | 日常护嗓习惯 | 否 | 用户手动填写 |
| LastScreenTime | LocalDateTime | 末次自查时间 | 否 | 自动更新 |
| HealthTrendTag | String(20) | 健康趋势标签 | 否 | 枚举：好转/稳定/恶化/无数据 |
| LatestDetectId | Integer | 关联最新检测ID | 否 | 外键，关联TAudioScreenRecord表 |
| RiskAssessmentLevel | Integer | 风险评估等级 | 否 | 0=低，1=中，2=高，null=未评定 |
| CreationTime | LocalDateTime | 创建时间 | 是 | 自动填充 |
| IsDelete | Boolean | 软删除标记 | 是 | 默认false |

**索引设计**：
- 主键索引：Id
- 唯一索引：UserId（确保每个用户只有一条档案）
- 普通索引：LatestDetectId

**业务约束**：
1. 每个用户只能有一条健康档案（UserId唯一）
2. RiskAssessmentLevel为null时，前端显示"待评定"
3. HealthTrendTag为空时，前端显示"暂无数据"

---

### 3.2 症状日志表（TSymptomLog）

**表结构**（基于现有表优化）：

| 字段名 | 类型 | 说明 | 必填 | 备注 |
|--------|------|------|------|------|
| Id | Integer | 主键ID | 是 | 自增 |
| UserId | Integer | 关联用户ID | 是 | 外键，关联AppUser表 |
| DetectId | Integer | 关联检测记录ID | 否 | 外键，关联TAudioScreenRecord表 |
| SymptomType | String(50) | 症状类型 | 是 | 枚举：嘶哑/疼痛/吞咽困难/咽干/咽痒/其他 |
| SymptomLevel | Integer | 症状严重程度 | 是 | 1=轻度，2=中度，3=重度 |
| SymptomDuration | String(50) | 症状持续时长 | 是 | 如"3天"、"2周" |
| SymptomOccurTime | LocalDateTime | 症状首次出现时间 | 否 | 默认当前时间 |
| Remark | String(500) | 备注说明 | 否 | 最多200字 |
| CreationTime | LocalDateTime | 创建时间 | 是 | 自动填充 |
| IsDelete | Boolean | 软删除标记 | 是 | 默认false |

**索引设计**：
- 主键索引：Id
- 普通索引：UserId（用于查询用户所有症状记录）
- 普通索引：DetectId（用于查询关联的检测记录）

**业务约束**：
1. SymptomType为"其他"时，Remark字段必须填写具体症状描述
2. SymptomLevel必须为1、2、3之一
3. 软删除的记录不参与风险等级计算

---

### 3.3 数据关联关系

```
AppUser (用户表)
  ├── TPersonalLaryngealHealthRecord (健康档案表) - 1:1
  │     └── TAudioScreenRecord (最新检测记录) - 1:1
  │
  ├── TSymptomLog (症状日志表) - 1:N
  │     └── TAudioScreenRecord (关联检测记录) - N:1
  │
  └── TAudioScreenRecord (检测记录表) - 1:N
```

#### 5.1.3 自动更新档案（检测完成后调用）

**调用位置**（在AudioDetectController.java中）：
```java
// 检测成功后，自动更新健康档案
if (pySuccess && userId != null && userId > 0) {
    try {
        // 查询或创建健康档案
        TPersonalLaryngealHealthRecordPagedInput queryInput = new TPersonalLaryngealHealthRecordPagedInput();
        queryInput.setUserId(userId);
        queryInput.setPage(1);
        queryInput.setLimit(1);
        TPersonalLaryngealHealthRecordDto healthRecord = TPersonalLaryngealHealthRecordService.Get(queryInput);
        
        if (healthRecord == null) {
            // 创建新档案
            healthRecord = new TPersonalLaryngealHealthRecordDto();
            healthRecord.setUserId(userId);
        }
        
        // 更新档案信息
        healthRecord.setLastScreenTime(LocalDateTime.now());
        healthRecord.setLatestDetectId(dto.getId()); // dto为刚创建的TAudioScreenRecordDto
        
        // 调用趋势计算和风险评定（见5.3节）
        healthRecord.setHealthTrendTag(healthTrendService.calculateHealthTrend(userId));
        healthRecord.setRiskAssessmentLevel(healthTrendService.calculateRiskLevel(userId));
        
        // 保存档案
        TPersonalLaryngealHealthRecordService.CreateOrEdit(healthRecord);
    } catch (Exception e) {
        // 档案更新失败不影响检测结果返回
        e.printStackTrace();
    }
}
```

## 六、业务流程

### 6.1 健康档案创建流程

```
用户完成首次检测
    ↓
检测状态变为"完成"
    ↓
调用 /TPersonalLaryngealHealthRecord/AutoUpdate
    ↓
检查是否存在健康档案
    ├─ 不存在 → 创建新档案
    └─ 存在 → 更新现有档案
    ↓
更新 LastScreenTime、LatestDetectId
    ↓
调用趋势计算服务，更新 HealthTrendTag
    ↓
调用风险评定服务，更新 RiskAssessmentLevel
    ↓
返回成功
```

---

### 6.2 症状记录流程

```
用户进入症状记录表单页
    ↓
填写症状信息（类型、程度、时长等）
    ↓
选择关联检测记录（可选）
    ↓
提交表单
    ↓
调用 /TSymptomLog/CreateOrEdit
    ↓
验证必填字段
    ↓
保存症状记录
    ↓
触发风险等级重新评定
    ↓
返回成功，跳转到症状列表页
```

---

### 6.3 风险等级评定流程

```
触发条件（检测完成/症状记录变更）
    ↓
获取用户最新检测结果
    ↓
获取用户最近症状记录
    ↓
计算检测结果得分（0-60分）
    ├─ 异常+高置信度 → 60分
    ├─ 异常+中置信度 → 40分
    ├─ 异常+低置信度 → 20分
    └─ 正常 → 0分
    ↓
计算症状严重程度得分（0-30分）
    ├─ 重度 → 30分
    ├─ 中度 → 15分
    └─ 轻度/无 → 0分
    ↓
计算持续时间得分（0-10分）
    ├─ >1个月 → +10分
    ├─ 1周-1个月 → 0分
    └─ <1周 → -5分（最低0分）
    ↓
综合得分 = 检测得分 + 症状得分 + 时长得分
    ↓
评定风险等级
    ├─ 得分<40 → 低风险（0）
    ├─ 得分40-70 → 中风险（1）
    └─ 得分>70 → 高风险（2）
    ↓
更新健康档案 RiskAssessmentLevel
```

```java
package com.example.web.service.impl;

import com.baomidou.mybatisplus.core.conditions.query.LambdaQueryWrapper;
import com.baomidou.mybatisplus.core.toolkit.Wrappers;

import java.util.regex.Pattern;
import java.util.regex.Matcher;

@Service
public class HealthTrendServiceImpl implements HealthTrendService {
    
    @Autowired
    private TAudioScreenRecordMapper audioScreenRecordMapper;
    
    @Autowired
    private TSymptomLogMapper symptomLogMapper;
    
    @Override
    public String calculateHealthTrend(Integer userId) {
        // 1. 查询最近3次有效检测记录（状态为完成）
        LambdaQueryWrapper<TAudioScreenRecord> queryWrapper = Wrappers
            .<TAudioScreenRecord>lambdaQuery()
            .eq(TAudioScreenRecord::getUserId, userId)
            .eq(TAudioScreenRecord::getDetectTotalStatus, true) // 只查询已完成的
            .eq(TAudioScreenRecord::getIsDelete, false)
            .orderByDesc(TAudioScreenRecord::getCreationTime)
            .last("LIMIT 3");
        
        List<TAudioScreenRecord> records = audioScreenRecordMapper.selectList(queryWrapper);
        
        // 2. 判断数据量
        if (records == null || records.size() < 3) {
            return "无数据";
        }
        
        // 3. 获取最近一次和前一次的结果
        TAudioScreenRecord latest = records.get(0);
        TAudioScreenRecord previous = records.get(1);
        
        // 4. 判定趋势
        Boolean latestResult = latest.getPrimaryScreenResult(); // true=异常，false=正常
        Boolean previousResult = previous.getPrimaryScreenResult();
        Double latestConfidence = latest.getPrimaryScreenConfidence();
        Double previousConfidence = previous.getPrimaryScreenConfidence();
        
        // 情况1：结果反转（异常→正常 = 好转，正常→异常 = 恶化）
        if (latestResult == false && previousResult == true) {
            return "好转";
        } else if (latestResult == true && previousResult == false) {
            return "恶化";
        }
        
        // 情况2：结果一致，看置信度变化
        if (latestResult == previousResult) {
            if (latestResult == true) {
                // 都是异常，置信度下降>10% = 好转
                if (previousConfidence != null && latestConfidence != null) {
                    double diff = previousConfidence - latestConfidence;
                    if (diff > 0.1) {
                        return "好转";
                    } else if (diff < -0.1) {
                        return "恶化";
                    }
                }
            } else {
                // 都是正常，置信度变化在±5%以内 = 稳定
                if (previousConfidence != null && latestConfidence != null) {
                    double diff = Math.abs(previousConfidence - latestConfidence);
                    if (diff <= 0.05) {
                        return "稳定";
                    }
                }
            }
        }
        
        // 默认返回稳定
        return "稳定";
    }
    
    @Override
    public Integer calculateRiskLevel(Integer userId) {
        int score = 0;
        
        // 1. 检测结果得分（0-60分）
        LambdaQueryWrapper<TAudioScreenRecord> detectQuery = Wrappers
            .<TAudioScreenRecord>lambdaQuery()
            .eq(TAudioScreenRecord::getUserId, userId)
            .eq(TAudioScreenRecord::getDetectTotalStatus, true)
            .eq(TAudioScreenRecord::getIsDelete, false)
            .orderByDesc(TAudioScreenRecord::getCreationTime)
            .last("LIMIT 1");
        
        TAudioScreenRecord latestDetect = audioScreenRecordMapper.selectOne(detectQuery);
        
        if (latestDetect != null && latestDetect.getPrimaryScreenResult() != null) {
            if (latestDetect.getPrimaryScreenResult()) {
                // 异常（恶性）
                Double confidence = latestDetect.getPrimaryScreenConfidence();
                if (confidence != null) {
                    if (confidence > 0.7) {
                        score += 60; // 高置信度
                    } else if (confidence > 0.5) {
                        score += 40; // 中置信度
                    } else {
                        score += 20; // 低置信度
                    }
                } else {
                    score += 40; // 默认中置信度
                }
            }
            // 正常（良性）不加分
        }
        
        // 2. 症状严重程度得分（0-30分）
        LambdaQueryWrapper<TSymptomLog> symptomQuery = Wrappers
            .<TSymptomLog>lambdaQuery()
            .eq(TSymptomLog::getUserId, userId)
            .eq(TSymptomLog::getIsDelete, false)
            .orderByDesc(TSymptomLog::getCreationTime)
            .last("LIMIT 1");
        
        TSymptomLog latestSymptom = symptomLogMapper.selectOne(symptomQuery);
        
        if (latestSymptom != null && latestSymptom.getSymptomLevel() != null) {
            Integer level = latestSymptom.getSymptomLevel();
            if (level == 3) {
                score += 30; // 重度
            } else if (level == 2) {
                score += 15; // 中度
            }
            // 轻度（1）不加分
        }
        
        // 3. 持续时间得分（-5到10分）
        if (latestSymptom != null && latestSymptom.getSymptomDuration() != null) {
            String duration = latestSymptom.getSymptomDuration();
            int durationDays = parseDurationToDays(duration);
            
            if (durationDays > 30) {
                score += 10; // >1个月
            } else if (durationDays < 7) {
                score = Math.max(0, score - 5); // <1周，最低0分
            }
            // 1周-1个月不加分
        }
        
        // 4. 评定等级
        if (score < 40) {
            return 0; // 低风险
        } else if (score < 70) {
            return 1; // 中风险
        } else {
            return 2; // 高风险
        }
    }
    
    /**
     * 解析持续时间字符串为天数
     * 如："3天" -> 3, "2周" -> 14, "1个月" -> 30
     */
    private int parseDurationToDays(String duration) {
        if (duration == null || duration.isEmpty()) {
            return 0;
        }
        
        Pattern pattern = Pattern.compile("(\\d+)(天|周|个月|月)");
        Matcher matcher = pattern.matcher(duration);
        
        if (matcher.find()) {
            int number = Integer.parseInt(matcher.group(1));
            String unit = matcher.group(2);
            
            switch (unit) {
                case "天":
                    return number;
                case "周":
                    return number * 7;
                case "个月":
                case "月":
                    return number * 30;
                default:
                    return 0;
            }
        }
        
        return 0;
    }
}
```

```javascript
            Page: 1,
            Limit: 1
        })
        
        if (response.Success && response.Data) {
            healthRecord.value = response.Data
            // 加载最新症状
            await loadLatestSymptom()
            // 加载趋势数据
            await loadTrendData()

        const hours = String(date.getHours()).padStart(2, '0')
        const minutes = String(date.getMinutes()).padStart(2, '0')
        return `${month}-${day} ${hours}:${minutes}`
    } catch (e) {
        return ''
    }
}
```

```java
private LambdaQueryWrapper<TPersonalLaryngealHealthRecord> BuilderQuery(
    TPersonalLaryngealHealthRecordPagedInput input) {
    
    LambdaQueryWrapper<TPersonalLaryngealHealthRecord> queryWrapper = Wrappers
        .<TPersonalLaryngealHealthRecord>lambdaQuery()
        .eq(input.getId() != null && input.getId() != 0, 
            TPersonalLaryngealHealthRecord::getId, input.getId())
        .eq(input.getUserId() != null && input.getUserId() != 0, 
            TPersonalLaryngealHealthRecord::getUserId, input.getUserId())
        .eq(input.getRiskAssessmentLevel() != null, 
            TPersonalLaryngealHealthRecord::getRiskAssessmentLevel, 
            input.getRiskAssessmentLevel())
        .eq(TPersonalLaryngealHealthRecord::getIsDelete, false);
    
    // 按创建时间倒序
    queryWrapper.orderByDesc(TPersonalLaryngealHealthRecord::getCreationTime);
    
    return queryWrapper;
}
```
