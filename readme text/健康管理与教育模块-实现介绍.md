## 健康管理与教育模块（科普 + 互动 + 风险问卷）- 实现介绍文档

## 一、模块概述

健康管理与教育模块围绕“**健康科普 + 用户互动 + 风险自评问卷**”三条主线，为居家喉部健康自查小程序提供持续的健康教育内容与自我管理工具。  

本模块已经完成以下几块能力：

- **科普浏览**：按分类浏览、关键字搜索、阅读去重统计、基础推荐/热度排序。
- **互动行为**：文章点赞、收藏、评论（支持楼中楼回复）、评论审核与软删除。
- **风险自评问卷**：用户完成风险问卷，系统计算得分与风险等级，同步到健康档案。
- **个人视角入口**：我的发布、我的收藏、最近测评结果摘要等。

后端基于 **Spring Boot + MyBatis-Plus**，前端基于 **uni-app（微信小程序/H5）**，管理端为 **Element Plus**。接口统一通过 `GlobalResponseAdvice` 封装为：

```json
{
  "Code": "200",
  "Success": true,
  "Msg": "成功",
  "Data": { ... }
}
```

## 二、主要实现功能概览

### 2.1 科普内容（文章）

- **后台管理**（ElementUI）
  - 列表、搜索、导出：`THealthScienceList.vue` + `THealthScienceController/THealthScienceServiceImpl`。
  - 分类管理：`THealthScienceCategoryList.vue` + `THealthScienceCategoryController/THealthScienceCategoryServiceImpl`。
  - 字段支持：封面图、富文本正文、摘要、展示状态、审核状态、置顶、推荐权重、互动计数（阅读/点赞/收藏/评论）。
- **前台展示**（uni-app）
  - 科普首页：`pages/Front/ScienceHome.vue`
    - 顶部分类 Tab（一级）+ 二级分类宫格。
    - 默认拉取推荐列表 `/Front/Science/List`。
  - 列表页：`pages/Front/ScienceList.vue`
    - 搜索栏（标题/关键字），按类型筛选（疾病科普/护喉贴士等）。
    - 支持分页上拉加载。
  - 详情页：`pages/Front/ScienceDetail.vue`
    - 展示标题、封面、阅读/点赞/收藏数量。
    - 富文本正文展示（后端存 HTML，前端用 `rich-text` 渲染）。
    - 进入详情即上报阅读 `/Front/Science/Read`，支持 30 分钟窗口去重。

### 2.2 互动功能（点赞 / 收藏 / 评论）

- **点赞/收藏**
  - 点赞表：`TScienceLike`，收藏表：`TScienceCollect`。
  - 唯一键（由 SQL 增量脚本提供）：防止重复点赞/收藏。
  - 通过 `/Front/Science/LikeToggle`、`/Front/Science/CollectToggle` 实现幂等操作。
- **评论与楼中楼**
  - 评论表：`TScienceComment`。
  - 字段：`ParentCommentId`（父评论）、`RootCommentId`（主楼）、`ReplyToUserId`（被回复用户）、`AuditStatus`、`IsDelete` 等。
  - 评论创建：`/Front/Science/CommentCreate`，支持主楼及子楼回复。
  - 评论列表：`/Front/Science/CommentList`，主楼分页 + 子回复按需展开。
  - 前台展示：`ScienceDetail.vue` 中展示主楼 + 子楼结构。

### 2.3 风险自评问卷

- **后台管理**
  - 问卷定义：`TRiskAssessmentQuestionnaire` + `TRiskAssessmentQuestionnaireController`。
  - 答案记录：`TUserQuestionnaireAnswer` + `TUserQuestionnaireAnswerController`（管理端查看历史）。
- **前台流程**
  - 问卷首页：`QuestionnaireHome.vue`
    - 展示最近一次测评结果（等级 + 分数 + 时间）。
    - 罗列可用问卷列表。
  - 答题页：`QuestionnaireAnswer.vue`
    - 根据现有后端模型实现一份“简化版风险问卷”（吸烟史、饮酒史、职业用嗓、反酸症状、其他风险因素）。
  - 结果页：`QuestionnaireResult.vue`
    - 展示分数、风险等级、文字说明。
    - 提供“去看科普”和“去设置提醒”的引导按钮。
- **档案联动**
  - 问卷提交 `/Front/Questionnaire/Submit` 后，写入 `TUserQuestionnaireAnswer`。
  - 同时调用 `TPersonalLaryngealHealthRecordMapper` 更新 `RiskAssessmentLevel`（健康档案表）。

### 2.4 个人视角功能

- **我的发布**：`MyScience.vue`
  - 通过 `/Front/Science/MyPublishList` 拉取当前用户作为作者的文章，支持按审核状态筛选（草稿/待审/通过/驳回/下架）。
  - 支持跳转到 `SciencePublish.vue` 继续编辑。
- **我的收藏**：`MyCollections.vue`
  - 通过 `/Front/Science/MyCollectList` 拉取当前用户收藏的文章。
  - 点击进入 `ScienceDetail.vue`。
- **最近测评摘要**：`UserCenter.vue`
  - 通过 `/Front/Questionnaire/LatestResult` 获取最近一次测评结果，在个人中心顶部展示“最近测评”卡片。

## 三、后端实现说明（Spring Boot）

### 3.1 核心实体与 Mapper

- 科普文章：`THealthScience` + `THealthScienceMapper`
  - 重要字段（增量脚本扩展后）：
    - `ScienceContent`（LONGTEXT，富文本）
    - `Summary`（摘要）
    - `AuditStatus/AuditorId/AuditTime/RejectReason/AuditRemark`
    - `PublishTime/IsTop/RecommendWeight`
    - `ReadCount/LikeCount/CollectCount/CommentCount`
- 科普分类：`THealthScienceCategory` + `THealthScienceCategoryMapper`
  - 支持多级分类：`ParentId/Level/Path`，展示状态 `ShowStatus`。
- 评论：`TScienceComment` + `TScienceCommentMapper`
  - 楼中楼关键字段：`ParentCommentId/RootCommentId/ReplyToUserId`。
  - 审核字段：`AuditStatus/AuditorId/AuditTime/RejectReason/AuditRemark`。
- 互动表：`TScienceLike/TScienceCollect/TScienceCommentLike` 及对应 Mapper。
- 阅读日志：`TScienceReadLog` + `TScienceReadLogMapper`
  - 来自脚本 `create_tsciencereadlog_table.sql`，用于 30 分钟窗口去重。
- 问卷与答案：`TRiskAssessmentQuestionnaire`、`TUserQuestionnaireAnswer`。
- 健康档案：`TPersonalLaryngealHealthRecord`。

### 3.2 前台业务 Service（Front 层）

#### 3.2.1 `FrontScienceService`

位置：`HealthControl.springboot/src/main/java/com/example/web/service/front/FrontScienceService.java`  
主要方法：

- `categoryList(FrontScienceCategoryListInput)`
  - 查询 `THealthScienceCategory`，按 `ParentId/SortNum` 组装成树结构。
  - `IncludeEmpty=false` 时，仅返回包含可见文章的分类（利用 `thealthscience` 过滤）。
- `list(FrontScienceListInput)`
  - 过滤条件：`AuditStatus=2 && ShowStatus=1 && IsDelete!=1`。
  - 支持：分类筛选（含子分类）、关键字搜索（标题+摘要）、类型筛选、排序（RECOMMEND/LATEST/HOT_7D/HOT_30D）。
  - 返回 `FrontScienceListItemDto`（Title/Cover/Summary/计数等）。
- `detail(FrontScienceDetailInput)`
  - 同样按“前台可见”条件过滤后返回 `THealthScienceDto`。
  - 根据当前登录用户计算 `IsLiked/IsCollected`。
- `read(FrontScienceReadInput)`
  - 计算 30 分钟窗口起始时间。
  - 尝试向 `TScienceReadLog` 插入一条（登录用户使用 UserId，未登录使用 ClientKey）。
  - 若唯一键冲突 → 视为去重，不增加阅读量；否则 `ReadCount+1`。
- `likeToggle(FrontScienceToggleInput)` / `collectToggle(FrontScienceToggleInput)`
  - 根据是否已有记录 + `IsDelete` 状态决定“点赞/取消点赞”最终状态。
  - 使用事务，并在唯一键冲突时回退为更新逻辑，保证幂等。
  - 同步冗余计数字段 `LikeCount/CollectCount`，避免列表频繁聚合。
- `commentCreate(FrontScienceCommentCreateInput)`
  - 校验评论内容长度（1–200 字）、文章可评论性。
  - 处理主楼/子楼的 `ParentCommentId/RootCommentId/ReplyToUserId`，并根据角色（管理员/普通用户）决定 `AuditStatus` 初始值。
  - 主楼插入后回填 `RootCommentId=自身 Id`。
- `commentList(FrontScienceCommentListInput)`
  - 若 `ParentCommentId` 为空 → 主楼分页：`ParentCommentId IS NULL OR 0`；按创建时间倒序。
  - 若有 `ParentCommentId` → 子楼列表：按创建时间正序。
  - 登录用户可看到自己尚未通过/被驳回的评论，他人仅见通过的评论。
  - 对主楼补齐：`ReplyCount` + `Children`（默认 2 条子回复预览，可由前端继续拉子回复）。
- `myPublishList(FrontScienceMyPublishListInput)` / `myCollectList(FrontScienceMyCollectListInput)`
  - 根据当前登录用户查询自己发布的文章 / 自己收藏的文章。
  - 返回同 `FrontScienceListItemDto` 结构，`myPublishList` 额外带回审核状态与驳回原因。
- `createOrEdit(FrontScienceCreateOrEditInput)` / `myDetail(...)`
  - 校验并保存作者发布的文章，支持“草稿（DRAFT）/提交审核（SUBMIT）”两种模式。
  - `myDetail` 用于作者侧回填编辑表单，仅允许查看/编辑自己的文章。

#### 3.2.2 `FrontQuestionnaireService`

位置：`HealthControl.springboot/src/main/java/com/example/web/service/front/FrontQuestionnaireService.java`  
主要方法：

- `list()`：返回所有 `ShowStatus=1 && IsDelete!=1` 的问卷。
- `detail(FrontQuestionnaireDetailInput)`：获取指定问卷。
- `submit(FrontQuestionnaireSubmitInput)`：
  - 根据问卷答案简单计算得分（吸烟/饮酒/职业用嗓/反酸/其他风险），封顶 100。
  - 映射风险等级：0–34=低、35–64=中、65–100=高。
  - 插入 `TUserQuestionnaireAnswer` 一条记录。
  - 更新/创建 `TPersonalLaryngealHealthRecord` 的 `RiskAssessmentLevel`。
- `myHistory(FrontPagedInput)`：返回当前用户最近 N 条问卷答题记录。
- `latestResult()`：返回当前用户最新一次问卷结果摘要（Id/Score/Level/AnswerTime）。

### 3.3 前台控制器

- `FrontScienceController`（`/Front/Science/*`）
  - `CategoryList/List/Detail/Read/LikeToggle/CollectToggle/CommentCreate/CommentList/MyPublishList/CreateOrEdit/MyDetail/MyCollectList`。
- `FrontQuestionnaireController`（`/Front/Questionnaire/*`）
  - `List/Detail/Submit/MyHistory/LatestResult`。

### 4.2 HTTP 封装与登录态

`HealthControl.uniapp/utils/http.js`：

- 统一 `Post` 方法：自动带上本地存储的 `Authorization: Bearer <token>`。
- 后端响应通过 `GlobalResponseAdvice` 统一封装，`Post` 返回结构：`{ Code, Success, Msg, Data }`。
- 使用 `PostSilent` 对于阅读上报这类不影响主流程的请求进行“静默调用”（失败不弹 toast）。

`store/index.js` 中的 `commonStore`：

- 提供 `Token/UserInfo/RoleType/UserId` 等计算属性。
- 封装 `CheckIsLogin/GetInfo/Logout` 等方法，被各前台页面复用。

## 五、小程序功能调试指南（重点）

下面按“用户真实路径”给出在小程序中调试本模块的具体步骤。  

### 5.1 基础准备

1. **确保后端已启动且数据库已执行增量脚本**：
   - `alter_thealthscience_upgrade_content_and_audit.sql`
   - `alter_thealthsciencecategory_add_hierarchy.sql`
   - `alter_tsciencecomment_extend_reply_and_audit_fields.sql`
   - `alter_science_interaction_add_unique_indexes.sql`
   - `create_tsciencereadlog_table.sql`
2. **确认 uni-app 环境指向正确后端**：
   - `HealthControl.uniapp/utils/config.js` 中的 `API_BASE_URL` 设置为你的后端地址（本地或 ngrok）。
3. **使用 HBuilderX 运行到微信小程序**：
   - 打开项目 → 运行 → 运行到小程序模拟器 → 微信开发者工具。
   - 或按《真机调试配置指南.md》通过 ngrok 做 HTTPS 隧道，在真机上调试。  

> 下文所有接口调试，都默认你已经能在小程序里正常登录（`/User/SignIn` 正常）。

### 5.2 科普模块调试

#### 5.2.1 科普首页与分类（ScienceHome.vue）

**目标**：验证分类树与推荐列表展示正常。  

1. **进入路径**：
   - 首页 → 点击“健康科普”卡片。
2. **预期行为**：
   - 顶部展示若干一级分类 Tab（如“声带保护”“疾病科普”等）。
   - 下方展示对应二级分类宫格（如“嗓音保健”“术后护理”等）。
   - 页面底部为推荐文章列表卡片。
3. **接口调试**：
   - 观察 Network（使用 H5 调试或微信开发者工具）：
     - `POST /Front/Science/CategoryList`：返回分类树。
     - `POST /Front/Science/List`：入参 `Page=1,Limit=10,Sort=RECOMMEND`。
4. **问题排查**：
   - 若分类为空：先在管理端 `THealthScienceCategory` 中新增分类，并确保 `ShowStatus=1`、`IsDelete=0`。
   - 若列表为空：在管理端新增一篇已通过审核的科普文章（见 5.2.4 审核说明）。

#### 5.2.2 列表筛选与搜索（ScienceList.vue）

**目标**：验证关键字搜索与类型筛选。  

1. **进入路径**：
   - 从科普首页顶部搜索框点击进入；
   - 或从二级分类宫格进入带 `CategoryId` 的列表。  
2. **调试步骤**：
   - 在搜索框输入关键词（如“护嗓”“喉炎”），点击键盘“搜索”。
   - 切换“类型”筛选，如：疾病科普/护喉贴士/就医指南等。
   - 上拉到底部，触发分页加载，看是否继续加载下一页。  
3. **接口观测**：
   - `POST /Front/Science/List`
     - 带上 `Keyword/KnowledgeType/CategoryId/Page/Limit/Sort` 等字段。
   - 注意列表为空但后端 `TotalCount>0` 时，检查分页逻辑与 Limit 设置。  

#### 5.2.3 详情、阅读计数与评论树（ScienceDetail.vue）

**目标**：验证文章详情、阅读去重、点赞/收藏、评论楼中楼。  

1. **进入路径**：
   - 列表页点击任意文章卡片进入详情。  
2. **阅读计数**：
   - 进入详情后，前端会调用：`POST /Front/Science/Read`，携带 `ScienceId + ClientKey`。
   - 在 30 分钟内反复进入同一文章，数据库 `ReadCount` 只应增加 1 次（可在管理端导出或直接查表确认）。  
3. **点赞/收藏**：
   - 登录状态下，点击底部“赞/藏”按钮：
     - 首次点击 → 点亮按钮，对应计数 +1。
     - 再次点击 → 取消，计数 -1。
   - 按钮连点数次，接口仍然返回成功，最终状态与点击次数奇偶一致，计数不会错乱（幂等）。  
4. **评论与楼中楼**：
   - 在详情底部输入评论 → 提交：
     - 普通用户：前台显示“审核中”，他人暂不可见。
     - 管理员用户：可直接通过审核（见管理端部分），前台立即可见。
   - 点击某条主楼的“回复” → 对该主楼或其子回复进行回复。
   - 点击“展开 X 条回复” → 调用 `CommentList` 带 `ParentCommentId` 拉取全部子回复。  
5. **接口观测**：
   - `POST /Front/Science/Detail`
   - `POST /Front/Science/Read`
   - `POST /Front/Science/LikeToggle`
   - `POST /Front/Science/CollectToggle`
   - `POST /Front/Science/CommentCreate`
   - `POST /Front/Science/CommentList`

### 5.3 作者发布与审核流程

#### 5.3.1 作者侧发布（SciencePublish.vue / MyScience.vue）

1. **进入路径**：
   - 个人中心 → “我的发布” → `MyScience.vue`。
   - 右上角“发布”按钮 → 进入 `SciencePublish.vue`。  
2. **发布步骤**：
   - 填写标题（5–40 字）、选择分类和知识类型。
   - 上传封面图（通过 `UploadImages` 组件，走 `/File/BatchUpload`）。
   - 填写摘要（可选）与正文（通过富文本组件 `rich-text-edit`）。  
3. **保存与提交**：
   - 点击导航右上角“草稿” → 调用 `/Front/Science/CreateOrEdit`，`SubmitType=DRAFT`。
   - 点击页面底部“提交审核” → `SubmitType=SUBMIT`，文章进入“待审核”状态。  
4. **查看发布状态**：
   - 返回“我的发布”，可按状态筛选：草稿/待审/通过/驳回/下架。
   - 对驳回的文章可看到驳回原因（来自后台审核填写的 `RejectReason`）。  

#### 5.3.2 管理端审核（THealthScienceList.vue）

1. 管理端登录 ElementUI 后台 → 菜单中进入“健康科普表”。  
2. 通过搜索条件（标题/作者/分类）查找待审文章。
3. 在编辑弹窗中查看正文内容：
   - 填写/修改 `AuditStatus`、`RejectReason`（当前版本使用原始 CRUD 字段管理；如需完整审核 UI，可在此基础上继续增强）。  
4. 审核通过后，前台列表/详情即可见该文章；下架/屏蔽状态文章前台不可见，但作者在“我的发布”中仍可看到状态。  

### 5.4 风险问卷与档案联动调试

#### 5.4.1 问卷列表与最近测评（QuestionnaireHome.vue）

1. **进入路径**：
   - 个人中心 → 风险自评 → `QuestionnaireHome.vue`。  
2. **预期行为**：
   - 顶部展示最近一次测评结果（风险等级 + 分数 + 时间）。
   - 下方列出可用问卷列表。  
3. **接口观测**：
   - `POST /Front/Questionnaire/LatestResult`
   - `POST /Front/Questionnaire/List`  

#### 5.4.2 答题与结果（QuestionnaireAnswer.vue / QuestionnaireResult.vue）

1. 在问卷首页选择一份问卷，进入 `QuestionnaireAnswer.vue`。
2. 填写各个问题选项后点击“提交测评”：
   - 前端弹出二次确认：“会同步更新健康档案风险等级”。
   - 调用 `POST /Front/Questionnaire/Submit`。
3. 提交成功后自动跳转到 `QuestionnaireResult.vue`：
   - 展示分数与风险等级（低/中/高）。
   - 展示文字说明和建议，并提供“去看科普”和“去设置提醒”的按钮。  
4. 返回个人中心，可看到“最近测评”卡片已更新。  
5. 进入“健康档案”页面，也可看到 `RiskAssessmentLevel` 已同步更新（参见《健康档案与筛查追踪模块-实现介绍.md》）。  

### 5.5 小程序端完整测试任务清单（仅用小程序，不看后台/数据库）

下面是一套**只通过小程序端即可完成的完整回归步骤**，按照顺序执行，可以覆盖本模块的所有核心功能。建议使用“正常用户账号”+“管理员账号”各跑一遍，以观察审核前后差异。

#### 任务 0：准备两个账号

1. 使用小程序中的“注册”入口创建两个账号：
   - A 号：普通用户（RoleType=2）。
   - B 号：管理员账号（可复用你原来的 `admin`，在后台已配置为角色 1）。  
2. 在手机或 H5 上确认两个账号都能登录成功。

> 后续步骤如无特殊说明，均在 **普通用户 A 号** 登录状态下操作。

#### 任务 1：科普浏览与阅读去重

1. 登录 A 号后，进入首页 → 点击“健康科普”卡片进入 `ScienceHome`。
2. 随机点击一个分类 Tab，观察下方二级分类是否变化。
3. 点击任一文章卡片进入详情页：
   - 记住当前显示的“阅读”数量（例如：阅读 10）。
4. 返回首页，再次进入同一篇文章详情：
   - 确认“阅读”计数在短时间（<30 分钟）内**不会无限增长**（可能 +1 后保持不变），说明去重逻辑生效。

#### 任务 2：列表搜索与筛选

1. 从 `ScienceHome` 顶部搜索框点击进入 `ScienceList` 页面。
2. 在搜索框输入一个关键词（例如“护嗓”）→ 回车或点击“搜索”：
   - 验证列表内容与关键词大致相关。
3. 打开“类型”选择器，依次选择“疾病科普”“护嗓贴士”等：
   - 确认列表会随类型变化而更新（如无数据，则显示空状态）。
4. 上拉列表到底部触发分页：
   - 若文章较多，观察“加载更多”逻辑是否正常；否则看到“没有更多了”提示。

#### 任务 3：点赞 / 收藏 / 并发幂等

1. 在任意一篇文章详情页（A 号登录）：  
   - 先记住当前“赞”“藏”数量。
2. 快速点击底部“赞”按钮 3~5 次：
   - 最终应停留在“亮起”状态，赞数比初始 +1 或保持合理，不会乱跳或变成异常大值。
3. 再快速点击“赞”按钮 3~5 次：
   - 最终应停留在“未点亮”状态，赞数比初始恢复或合理减 1，不会变成负数。
4. 按同样方式测试“收藏”按钮：
   - 验证状态与计数稳定，不会出现“状态已变，但数值没有同步”的问题。

#### 任务 4：评论与楼中楼（不看后台审核）

1. 在文章详情底部点击“评论”按钮，输入一条内容，例如“这篇很实用”：  
   - 提交后，应在列表顶部看到自己的评论，状态标记为“审核中”或“已发布”（视你是否使用管理员权限登录）。  
2. 点击该评论的“回复”按钮，输入“@自己 一句回复”，提交：  
   - 在该评论下方应出现子回复，或在“展开 X 条回复”中可以看到。  
3. 再对某条子回复点击“回复”，验证二级楼中楼展示是否清晰，不会混乱。  
4. 刷新详情页（返回再进）：
   - 确认自己的评论和回复仍然存在，状态标签正确（审核中/驳回/通过）。  

> 若要验证“通过/驳回/屏蔽”效果，可切换到管理员 B 号，在管理端网页审核，然后在小程序 A 号再次检查：  
> - 被驳回的评论仅作者在详情页看到“已驳回”字样，他人看不到。  

#### 任务 5：我的收藏

1. 在任意文章详情页（A 号）点击“收藏”，确保处于已收藏状态。
2. 回到个人中心 → 点击“我的收藏”：
   - 列表中应出现刚刚收藏的文章标题和封面。
3. 点击该收藏项进入详情页，再次点击“取消收藏”：  
   - 回到“我的收藏”，列表应更新，不再显示该文章。

#### 任务 6：我的发布 + 发布草稿 + 提交审核

1. 在个人中心 → 点击“我的发布”进入 `MyScience` 页面：
   - 如当前为空，显示提示“暂无发布内容”。
2. 点击右上角“发布”进入 `SciencePublish` 页面：
   - 填写标题（例如“护嗓日常小贴士”）。
   - 选择一个科普分类和“知识类型”。
   - 上传一张封面图片。
   - 在富文本中输入一段正文（可以简单输入几句话）。
3. 点击顶部导航右侧“草稿”：
   - 应弹出“保存成功”或类似提示，并返回“我的发布”。
   - 在列表中可以看到该文章状态为“草稿”。
4. 再次点击这条草稿进入编辑页，补充/修改内容后，点击底部“提交审核”：
   - 返回“我的发布”后，状态变为“待审核”。
5. 可以多创建一篇文章，用于后续管理员审核测试（可选）。

> 若你暂时不使用后台审核，也可以直接把“待审核”的文章当作“自己的草稿空间”使用，前台他人仍看不到未通过审核的文章。

#### 任务 7：风险自评问卷完整闭环

1. 在个人中心顶部，若尚未有测评记录，“最近测评”区域会提示“暂无测评”。
2. 点击菜单中的“风险自评”，进入 `QuestionnaireHome`：
   - 顶部若有最近结果，会显示分数与等级；初次使用则为空。  
3. 在“可用问卷”列表中选择一份问卷，进入 `QuestionnaireAnswer`：
   - 依次选择“吸烟史/饮酒史/职业用嗓/反酸症状”等选项，填写“其他风险因素”（可填“经常熬夜”）。
4. 点击“提交测评”，在确认弹窗中点击“确定”：
   - 页面跳转至 `QuestionnaireResult`，展示本次得分与风险等级，并有“去看科普/去设置提醒”的按钮。
5. 从结果页返回个人中心：
   - 顶部“最近测评”卡片应出现刚才的分数与风险等级。  
6. 若你再做一次测评（改用不同风险组合），可看到：
   - “最近测评”卡片会更新为最新一次数据。

#### 任务 8：科普与问卷的联合体验（推荐使用真实用户路径）

1. A 号登录后，先完成一次风险自评，确认个人中心展示“中风险”或“高风险”。
2. 再进入“健康科普”，观察首页推荐是否包含更多“疾病科普/就医指南”类文章（当前版本是基础策略，但后续可以按风险标签优化推荐）。
3. 选择一篇与你风险等级相关的科普文章（例如“高风险时的就医建议”），完成：
   - 阅读 → 点赞 → 收藏 → 评论与楼中楼。  
4. 在“我的收藏”与“我的发布”中，确认所有行为与状态均正确。  

完成以上 8 个任务后，你已经在**仅使用小程序端**的情况下，覆盖并验证了健康管理与教育模块实现的：科普展示、互动行为、作者发布、评论楼中楼、阅读去重、风险问卷与档案联动、个人视角入口等全部核心功能。

## 六、常见问题与排查

### 6.1 列表没有数据

- 检查是否有通过审核的文章：`AuditStatus=2 && ShowStatus=1 && IsDelete=0`。
- 检查增量 SQL 是否执行，特别是 `alter_thealthscience_upgrade_content_and_audit.sql`。

### 6.2 点赞/收藏状态异常

- 确认 `alter_science_interaction_add_unique_indexes.sql` 已执行，唯一键生效。
- 如计数与实际不一致，可通过导出或临时 SQL 校正 `LikeCount/CollectCount`。  

### 6.3 评论看不到或为“审核中”

- 普通用户新发评论默认是待审核状态，只有自己可见。
- 管理端审核通过后，他人才可见；若审核状态为“驳回/屏蔽”，则仅作者在“我的发布/评论管理”中看到原因。  

### 6.4 问卷提交不更新档案

- 确认 `FrontQuestionnaireService.submit` 中对 `TPersonalLaryngealHealthRecord` 的更新逻辑已经执行（查看后端日志）。
- 检查数据库中是否存在该用户的档案记录，如无则应由提交问卷时自动创建。
