# 小程序图片插入位置整理

## 1. 结论概览

结合项目代码、`healthcontrol.sql`、以及前台页面结构，目前小程序里的图片位可以分成 3 类：

| 类别 | 数量 | 说明 |
| --- | --- | --- |
| 需要人工上传/维护的图片位 | 6 类主图片位 + 1 类临时上传图 | 头像、科普封面、科普正文插图、食物配图、医生/医院配图、喉镜照片，以及 AI 图谱上传页的临时图片 |
| 系统自动生成的图片位 | 3 类 | 波形图、MFCC 图、Mel 图 |
| SQL 已预留但当前前台未真正使用的图片位 | 2 类 | 科普分类图标、就诊准备报告封面图 |

如果你后续是要安排“补图/出图/整理素材”，优先级最高的是：

1. 用户头像
2. 喉镜照片
3. 科普封面图
4. 科普正文插图
5. 饮食食物配图
6. 医院/医生头像或门诊配图

---

## 2. 需要人工上传或维护的图片位

### 2.1 用户头像

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 用户头像 |
| SQL 字段 | `appuser.ImageUrls` |
| 插图方式 | 用户自行上传 |
| 主要入口 | `pages/Front/UserInfoEdit.vue` |
| 展示页面 | `pages/Front/UserCenter.vue`、`pages/Front/HealthRecord.vue` |
| 当前状态 | 已接入，且编辑页把头像设成必填项 |
| 备注 | 无头像时前台会回退到默认头像 `HealthControl.uniapp/assets/默认头像.png` |

说明：

- 这是当前最明确、最稳定的用户自定义图片位。
- 头像会直接影响个人中心和健康档案页的视觉完整度。

### 2.2 喉镜照片

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 医院检查后的喉镜影像 |
| SQL 字段 | `tlaryngoscopephoto.LaryngoscopePhotoUrl` |
| 插图方式 | 用户上传 |
| 主要入口 | `pages/Front/LaryngoscopePhotoEdit.vue` |
| 展示页面 | `pages/Front/LaryngoscopePhotoList.vue`、`pages/Front/HealthRecord.vue`、`pages/Front/ModelLabelDetail.vue` |
| 关联入口 | `pages/Front/AudioResult.vue` 可跳转“上传喉镜” |
| 当前状态 | 已完整接入上传、列表、预览、编辑、删除 |
| 备注 | 1 条记录主打一张图，且可关联某次音频检测记录 |

说明：

- 这是小程序里最重要的医疗图片位之一。
- 后端生成就诊准备报告时，也会把喉镜照片嵌入报告 HTML/PDF。

### 2.3 科普封面图

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 科普文章封面 |
| SQL 字段 | `thealthscience.CoverUrl` |
| 插图方式 | 发布者/运营上传 |
| 主要入口 | `pages/Front/SciencePublish.vue` |
| 展示页面 | `pages/Front/ScienceHome.vue`、`pages/Front/ScienceList.vue`、`pages/Front/MyScience.vue`、`pages/Front/MyCollections.vue` |
| 当前状态 | 已接入，且前后端都把封面当成核心内容字段 |
| 备注 | 详情页 `ScienceDetail.vue` 当前主要展示正文，不单独重复展示封面 |

说明：

- 这是内容模块最需要补图的地方。
- 发布页文案明确提示“推荐使用横图”，适合按 16:9 思路准备。

### 2.4 科普正文插图

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 科普正文中的说明图、步骤图、示意图 |
| SQL 字段 | `thealthscience.ScienceContent`（富文本 HTML 内嵌 `<img>`） |
| 插图方式 | 发布者在富文本编辑器内插入图片 |
| 主要入口 | `components/rich-text-edit/rich-text-edit.vue` 的“插入图片”能力 |
| 展示页面 | `pages/Front/ScienceDetail.vue` |
| 当前状态 | 已接入 |
| 备注 | 这是正文级图片位，不是封面位 |

说明：

- 这类图最适合做“发音动作示意”“护嗓步骤图”“喉部结构说明图”。
- 从项目结构看，正文图片会跟正文一起存到 `ScienceContent` 里。

### 2.5 饮食食物配图

| 项目 | 内容 |

| 图片内容 | 食物卡片图 |
| SQL 字段 | `tlaryngealfood.PicUrl` |
| 插图方式 | 后台维护 |
| 主要入口 | 管理端/模板表单已有 `PicUrl` 上传位 |
| 展示页面 | `pages/Front/DietFoodList.vue`、`pages/Front/DietFoodDetail.vue` |
| 当前状态 | 已接入前台展示 |
| 备注 | 无图时前台会尝试回退到默认占位图 `/static/food-placeholder.png` |

说明：

- 这类图会直接影响饮食模块的完成度。
- 如果后续要做“推荐食物卡片”或“忌口卡片”，这一类素材需求会很集中。

### 2.6 医院/医生头像 / 门诊配图

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 医生头像、门诊配图、医院就诊形象图 |
| SQL 字段 | `totolaryngologyhospitaldoctor.PicUrl` |
| 插图方式 | 后台维护 |
| 主要入口 | 管理端/模板表单已有 `PicUrl` 上传位 |
| 展示页面 | `pages/Front/HospitalDoctorList.vue`、`pages/Front/HospitalDoctorDetail.vue`、`pages/Front/MedicalRecommendDetail.vue` |
| 当前状态 | 已接入前台展示 |
| 备注 | 无图时会显示人像占位，不会导致页面报错 |

说明：

- 这类图不是强制项，但会明显影响“就医辅助”模块的可信度和完成度。
- 当前 SQL 示例数据里已经放了示例图片 URL。

### 2.7 AI 图谱上传页的临时上传图片

| 项目 | 内容 |
| --- | --- |
| 图片内容 | 用户手动上传的 Mel / MFCC 图谱 |

| 当前状态 | 已接入 |

说明：

- 这个图片位不属于长期内容资产，但确实是小程序里一个明确的图片上传入口。
- 更适合记作“临时分析图”，不算内容运营补图位。

### 3.2 MFCC 图

| 项目 | 内容 |

### 3.3 Mel 图

| 项目 | 内容 |
| --- | --- |
| SQL 字段 | `taudioscreenrecord.MelSpectrumUrl` |
| 生成方式 | 音频检测流程自动生成 |
| 展示页面 | `pages/Front/AudioResult.vue`、`pages/Front/AudioAIReport.vue`、`pages/Front/AudioAIToolResult.vue` |

## 4. SQL 预留了字段，但当前前台没有真正用起来的图片位

### 4.1 科普分类图标

| 项目 | 内容 |
| --- | --- |
| SQL 字段 | `thealthsciencecategory.IconUrl` |
| 当前状态 | 后端 DTO 和接口已支持，但前台分类页暂未渲染该图标 |
| 现有前台表现 | `ScienceHome.vue`、`ScienceList.vue` 目前只显示分类文字 |
| 建议 | 如果后续要做更强内容首页，可补“分类 icon”素材 |

说明：

- 这个字段是“已经设计过，但前台还没吃掉”的典型图片位。
- 如果后续你想把科普首页做得更像内容频道，这一项值得启用。

### 4.2 就诊准备报告封面图

| 项目 | 内容 |
| --- | --- |
| SQL 字段 | `tusermedicalpreparereport.CoverImageUrl` |
| 当前状态 | 数据库字段已存在，但当前小程序前台没有单独展示这个封面图 |
| 现有前台表现 | `MedicalPrepareReport.vue` 目前只展示报告标题、状态、查看/下载动作 |

## 5. 不建议混进“补图清单”的静态素材

下面这些图片已经属于项目静态资源，不属于业务数据位，不建议和“需要插入的业务图片”混在一起统计：

- `HealthControl.uniapp/assets/logo.png`
- `HealthControl.uniapp/assets/默认头像.png`
- `HealthControl.uniapp/assets/custom-icons/*`
- `HealthControl.uniapp/assets/icons/*`
- 根目录 `icon/*`

这些更像是：

- 小程序固定图标
- Tab 图标
- 首页功能入口图标
- 默认占位资源

它们不是后续运营录入数据时要补的“内容图位”。

## 6. 最终建议：如果现在要补图，按这个顺序推进

### 第一批：最影响用户观感

1. 科普封面图
2. 饮食食物配图
3. 医院/医生配图

### 第二批：最影响医疗资料完整度

1. 用户头像
2. 喉镜照片

### 第三批：内容质量提升项

1. 科普正文插图
2. 科普分类图标（当前前台未启用，可后续启用）
3. 就诊准备报告封面图（当前前台未启用，可后续启用）
