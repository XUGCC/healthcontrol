## 就医辅助模块（医院医生库 + 就医建议 + 就诊准备报告）- 实现介绍文档

## 一、模块概述

就医辅助模块围绕“**自查之后，如何更好地线下就诊**”这一核心场景，为小程序用户提供：  

- 医院/医生信息查询与拨号、预约链接跳转；  
- 基于风险等级与自查结果的就医建议浏览；  
- 一键生成、回看与导出的就诊准备报告；  
- 导出记录留痕，形成“医疗沟通档案”的一部分。  

当前版本已完成 **前台（UniApp）+ 后台（Spring Boot）+ 管理端（ElementUI）** 的闭环实现，满足需求文档《就医辅助模块-需求.md》中定义的 **MVP 验收标准（8.1 / 8.2 / 8.3）**。

## 二、主要实现功能概览

### 2.1 就医辅助首页（MedicalHome）

- **页面**：`pages/Front/MedicalHome.vue`  
- **后端接口**：`POST /Front/Medical/Home`（`FrontMedicalController.home` → `FrontMedicalService.home`）  
- **主要能力**：
  - 汇总当前用户的就医相关关键信息：
    - `RiskLevel`：来自 `TPersonalLaryngealHealthRecord.RiskAssessmentLevel`，0/1/2=低/中/高风险；
    - `LatestDetect`：最近一次音频自查摘要（时间、结果、置信度等，简要文本）；
    - `LatestSymptom`：最近一条症状记录摘要（症状类型/持续时间等，前端文案展示）；
    - `LatestRecommend`：最近一条就医建议摘要（标题、推荐原因、时间、已读状态）；
    - `LatestReport`：最近一份就诊准备报告摘要（标题、状态文案、ReportId、Html/PDF 链接）；
    - `LatestVisitPlan`（预留）：就诊计划简要信息（计划日期、医生、渠道等）。
  - 首页卡片结构：
    - **当前风险概览**：展示风险等级 + 最近检测/症状摘要 + 风险说明文案；
    - **系统就医建议**：若存在最近推荐，展示标题、推荐原因、时间与“已查看/未查看”标记，点击进入详情或列表；
    - **就诊准备报告**：展示最近一份报告的标题和生成状态；支持一键生成新报告或打开最近报告；
    - **医院/医生库入口**：统一入口跳转到医院/医生列表页。
  - 未登录场景：
    - 后端允许匿名访问 `/Front/Medical/Home`，返回非个人化的默认结构；
    - 前端根据 `commonStore.UserId` 是否存在，决定是否展示用户个人数据说明。

### 2.2 医院/医生库（HospitalDoctorList / Detail）

- **页面**：
  - 列表页：`pages/Front/HospitalDoctorList.vue`
  - 详情页：`pages/Front/HospitalDoctorDetail.vue`
- **后端接口**：
  - `POST /Front/Medical/DoctorList`（列表）
  - `POST /Front/Medical/DoctorDetail`（详情）
- **核心实现**：
  - 列表查询 `TOtolaryngologyHospitalDoctor`，过滤条件：
    - 仅返回 `ShowStatus = 1` 且 `IsDelete != 1` 的记录（兼容历史 NULL）；
    - 支持 `Region`（地区）、`DepartmentType`（科室类型 0/1/2）、`Keyword`（医院名/医生名模糊匹配）筛选；
    - 分页返回：`Items + TotalCount`。
  - 列表卡片字段：
    - 医院名称、医生姓名；
    - 科室类型（映射为“耳鼻喉科/咽喉科/头颈外科”中文文案）；
    - 地区（Region）、电话（ContactPhone）、预约链接（AppointmentUrl）的可用性提示。
  - 交互：
    - 点击电话：前端调用 `uni.makePhoneCall`；
    - 点击预约链接：
      - 弹窗二次确认（提示将打开外部链接/复制链接）；
      - 根据小程序环境，打开浏览器 / 复制到剪贴板。
    - 点击卡片：进入 `HospitalDoctorDetail`，展示更完整信息（包括擅长方向、门诊时间等增强字段，如数据库中已补齐）。

### 2.3 就医推荐（MedicalRecommendList / Detail）

- **页面**：
  - 列表页：`pages/Front/MedicalRecommendList.vue`
  - 详情页：`pages/Front/MedicalRecommendDetail.vue`
- **后端接口**：
  - `POST /Front/Medical/RecommendList`
  - `POST /Front/Medical/RecommendDetail`
  - `POST /Front/Medical/RecommendMarkViewed`
  - （预留）`POST /Front/Medical/RecommendFeedback`
- **主要能力**：
  - 列表：
    - 返回当前登录用户的 `TUserMedicalRecommend` 列表；
    - 支持按 `ViewStatus`（未查看/已查看）筛选；
    - 列表项包含：推荐时间、风险等级/标签、推荐标题与简要原因、是否已查看标记。
  - 详情：
    - 打开详情即调用 `RecommendDetail` 获取完整信息，同时调用 `RecommendMarkViewed` 设置 `ViewStatus = 1`；
    - 展示推荐依据摘要：
      - 风险等级快照（`RiskLevel` 或 `PrimaryScreenResult/Confidence`）；
      - 关联的检测记录（`DetectId` → 音频自查摘要）；
      - 症状快照（如 `SymptomSnapshot` 文本）；
      - 后端生成的 `RecommendReason` 文案（支撑“为什么建议就医”）。
    - 展示推荐医院/医生卡片（如存在 `RecommendHospitalId`）：
      - 医院名称、医生姓名、科室、地区、电话、预约链接；
      - 点击电话拨打、点击链接跳转。
    - 文案规范：
      - 对不同风险等级做差异化提示（低风险偏“如持续不适建议就医”，高风险偏“尽快就医/复查”）；
      - 固定展示“本建议仅供参考，不能替代医生诊断”与“急诊提示”。
  - 反馈预留：
    - 前端结构与 DTO 中已预留 `Feedback` 字段；
    - 可在后续版本接入 `RecommendFeedback` 接口，用于记录“已就医/暂不需要”等反馈，不影响当前 MVP。

### 2.4 就诊准备报告（MedicalPrepareReport + MedicalReportWebView）

- **页面**：
  - 报告列表/生成页：`pages/Front/MedicalPrepareReport.vue`
  - HTML 报告预览页：`pages/Front/MedicalReportWebView.vue`（`web-view`）
- **后端接口**：
  - `POST /Front/Medical/PrepareReportCreate`
  - `POST /Front/Medical/PrepareReportList`
  - `POST /Front/Medical/PrepareReportDetail`
  - `GET /Front/Medical/PrepareReportHtml`（返回 HTML）
  - `GET /Front/Medical/PrepareReportPdf`（返回 PDF 二进制流）
- **核心实现**：
  - 报告实体：`TUserMedicalPrepareReport`
    - 新增字段：`PrimaryDetectId, UpdateTime, ReportTitle, ReportNo, StandardDocType, ReportContentJson, HtmlDocUrl, PdfDocUrl, CoverImageUrl` 等；
    - `GenerateStatus`：0 待生成 / 1 生成中 / 2 已完成 / 3 失败；
    - 统计字段：`ViewCount, DownloadCount, ShareCount`。
  - 报告创建 `prepareReportCreate`：
    - 根据当前用户最近的 `TPersonalLaryngealHealthRecord` 与 `TAudioScreenRecord/TSymptomLog/TUserDietRecord` 生成一份“内容快照 JSON”写入 `ReportContentJson`：
      - 包含本次报告对应的检测 ID 列表、症状 ID 列表、饮食记录 ID 列表、风险等级等；
      - 便于后续在历史数据改变时仍能还原当时报告视图。
    - 设置 `ReportTitle`（如“就诊准备报告（简版）”）、`ReportNo`（可选流水号）、`StandardDocType="HTML"`；
    - 调用内部方法 `buildReportHtml(...)` 生成 HTML 字符串，并写入 `ReportContentJson`（兼容老版本直接存 HTML 的情况）；
    - 构造 `HtmlDocUrl` / `PdfDocUrl` 指向 `PrepareReportHtml` / `PrepareReportPdf` 两个 GET 接口，供前端使用。
  - HTML 渲染 `prepareReportHtml`：
    - 读取数据库中 `ReportContentJson`：
      - 若内容以 `<` 开头，视为直接存储的 HTML，原样返回；
      - 否则按 JSON 结构解析出关联的检测/症状/饮食记录 ID，联表查询真实数据；
    - 调用 `buildReportHtml`：
      - 组装包含“报告抬头 + 风险等级 + 近期检测摘要 + 症状摘要 + 饮食摘要 + 温馨提示”的标准 HTML 页面；
      - 页面样式采用内联 CSS，保证在小程序 `web-view` 与 PDF 引擎中展示一致；
      - 固定插入免责声明与急重症就医提示。
    - `FrontMedicalController.PrepareReportHtml` 使用 `produces = "text/html;charset=UTF-8"`，直接返回 HTML 字符串。
  - PDF 导出 `prepareReportPdf`：
    - 使用与 `prepareReportHtml` 相同逻辑获取 HTML 源文档；
    - 通过 `openhtmltopdf-pdfbox` + `openhtmltopdf-slf4j` 将 HTML 渲染为 PDF（流写入 `HttpServletResponse`）；
    - 设置响应头 `Content-Type=application/pdf` 与合适的 `Content-Disposition`，支持浏览器直接预览或下载。
  - 前端交互：
    - 报告页面支持选择模板类型（简版/详版）后，点击“生成报告”调用 `PrepareReportCreate`；
    - 列表展示历史报告条目（标题、生成时间、状态、查看次数等），点击可打开详情；
    - 打开 HTML 报告：`uni.navigateTo` 到 `MedicalReportWebView`，使用 `HtmlDocUrl` 作为 `web-view` 源；
    - 下载 PDF 报告：前端使用 `PdfDocUrl` 调用 `uni.downloadFile`，并可继续通过 `uni.openDocument` 打开。

### 2.5 导出记录与可追溯（MedicalExportRecordList）

- **页面**：`pages/Front/MedicalExportRecordList.vue`  
- **后端接口**：
  - `POST /Front/Medical/ExportRecordCreate`
  - `POST /Front/Medical/ExportRecordList`
- **实现要点**：
  - 实体：`TScreenReportExportRecord`
    - 补充字段：`ReportId, ExportPurposeText, SourceType`（根据增量 SQL 脚本）；
    - 与 `TUserMedicalPrepareReport` 通过 `ReportId` 精确关联；
    - 兼容旧字段 `DetectId`。
  - 创建导出记录：
    - 在报告 HTML/PDF 预览或下载动作中，由前端调用 `ExportRecordCreate`；
    - 入参包含：`ReportId, DetectId(可选), ExportFormat, ExportPurpose, ExportFileUrl(可选)`；
    - 后端写入导出记录，并根据需要更新 `ViewCount/DownloadCount/ShareCount`。
  - 列表：
    - `ExportRecordList` 返回当前用户的导出记录，支持分页；
    - 前端展示导出时间、导出格式、用途（枚举 + 文本）、文件 URL（如有），作为“历史导出记录”的可视化。

## 三、后端实现说明（Spring Boot）

### 3.1 核心实体与 Mapper

- 医院医生库：`TOtolaryngologyHospitalDoctor`  
  - 关键字段：`HospitalName, DoctorName, DepartmentType, Region, Address, ContactPhone, AppointmentUrl, ShowStatus, IsDelete`  
  - 增强字段（来自增量 SQL）：`HospitalLevel, Tags, SpecialtyDesc, Longitude, Latitude, OutpatientTime, DoctorTitle, PicUrl, UpdateTime` 等。
- 就医推荐记录：`TUserMedicalRecommend`  
  - 字段：`UserId, DetectId, RiskLevel, RecommendHospitalId, RecommendTime, ViewStatus, IsDelete`  
  - 增强字段：`PrimaryScreenResult, PrimaryScreenConfidence, SymptomSnapshot, RecommendReason, FeedbackType, FeedbackRemark` 等。
- 就诊准备报告：`TUserMedicalPrepareReport`  
  - 字段：`UserId, HealthRecordId, DetectIdList, TemplateType, ContentScope, GenerateStatus, GenerateFailReason, ViewCount, DownloadCount, ShareCount, IsDelete`  
  - 增强字段：`PrimaryDetectId, UpdateTime, ReportTitle, ReportNo, StandardDocType, ReportContentJson, HtmlDocUrl, PdfDocUrl, CoverImageUrl`。
- 导出记录：`TScreenReportExportRecord`  
  - 字段：`UserId, DetectId, ExportTime, ExportFormat, ExportFileUrl, ExportPurpose, IsDelete`  
  - 增强字段：`ReportId, ExportPurposeText, SourceType`。
- 健康档案：`TPersonalLaryngealHealthRecord`  
  - 提供 `RiskAssessmentLevel, HealthTrendTag, LatestDetectDto` 等字段，为就医辅助首页与报告内容提供基础数据。

### 3.2 前台业务 Service（`FrontMedicalService`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/front/FrontMedicalService.java`  
主要方法：

- `home(FrontMedicalHomeOutput)`  
  - 根据当前登录用户 `UserId`：
    - 查询最新健康档案，映射风险等级与趋势文本；
    - 查询最近音频自查记录，构造 `LatestDetect` 简要 DTO；
    - 查询最近症状记录，构造 `LatestSymptom`；
    - 查询最近一条 `TUserMedicalRecommend` 与最近一条 `TUserMedicalPrepareReport`，构造首页展示所需字段；
    - 可选：查询最近就诊计划记录。
- `doctorList(FrontMedicalDoctorListInput)`  
  - 按 `Region / DepartmentType / Keyword` 分页查询 `TOtolaryngologyHospitalDoctor`；
  - 过滤 `ShowStatus=1 && IsDelete!=1`，按一定排序返回（如按更新时间/自定义排序号）。
- `doctorDetail(FrontMedicalDoctorDetailInput)`  
  - 根据 `DoctorId` 返回医生详情（含地图与预约相关字段）；
  - 若记录不存在或 `IsDelete=1`，抛出业务异常。
- `recommendList(FrontMedicalRecommendListInput)`  
  - 只返回当前用户的推荐记录（`UserId` 来自 `BaseContext`），按时间倒序分页；
  - 支持按 `ViewStatus` 过滤。
- `recommendDetail(FrontMedicalRecommendDetailInput)`  
  - 校验推荐记录归属（防越权）；
  - 组装推荐详情 DTO：包含风险快照、检测摘要、症状快照、推荐原因文案、推荐医生卡片等。
- `recommendMarkViewed(FrontMedicalRecommendMarkViewedInput)`  
  - 将指定推荐记录设置为已查看（`ViewStatus=1`），并记录更新时间。
- `recommendFeedback(FrontMedicalRecommendFeedbackInput)`（预留）  
  - 写入 `FeedbackType/FeedbackRemark` 字段，为后续策略优化提供基础。
- `prepareReportCreate(FrontMedicalPrepareReportCreateInput)`  
  - 创建 `TUserMedicalPrepareReport`，写入快照 JSON 与 HTML 文本，并设置 `HtmlDocUrl/PdfDocUrl`；
  - 根据创建结果返回报告 ID 与基础信息。
- `prepareReportList(FrontMedicalPrepareReportListInput)`  
  - 返回当前用户历史报告列表（分页），过滤 `IsDelete!=1`；
  - 列表项包含标题、生成时间、状态、查看/导出计数等。
- `prepareReportDetail(FrontMedicalPrepareReportDetailInput)`  
  - 查询指定 `ReportId`，校验用户归属，返回包含链接与快照内容在内的完整 DTO。
- `exportRecordCreate(FrontMedicalExportRecordCreateInput)`  
  - 幂等性与基础校验后，为当前用户插入一条导出记录；
  - 可选累加对应报告的 `DownloadCount/ShareCount`。
- `exportRecordList(FrontMedicalExportRecordListInput)`  
  - 返回当前用户的导出记录列表。

内部辅助方法：

- `buildReportHtml(TUserMedicalPrepareReport report, TPersonalLaryngealHealthRecord record, ...)`  
  - 将报告标题、生成时间、风险等级、近期检测/症状/饮食摘要与固定提示文案拼装成统一风格的 HTML 页面；
  - 兼容老版本直接存 HTML 的情况。
- `resolveReportHtmlFromJson(...)`  
  - 从 `ReportContentJson` 解析出 ID 列表，并查询真实数据，拼装内容块。

### 3.3 前台控制器（`FrontMedicalController`）

位置：`HealthControl.springboot/src/main/java/com/example/web/controller/FrontMedicalController.java`  
主要职责：

- 暴露 `/Front/Medical/*` 前台 REST 接口：
  - `Home / DoctorList / DoctorDetail / RecommendList / RecommendDetail / RecommendMarkViewed / RecommendFeedback / PrepareReportCreate / PrepareReportList / PrepareReportDetail / ExportRecordCreate / ExportRecordList`；
  - 统一封装响应为 `{ Code, Success, Msg, Data }`；
  - 使用 `BaseContext.getCurrentId()` 获取当前登录用户，所有涉及用户数据的接口做归属校验；
  - 对 `PrepareReportHtml` / `PrepareReportPdf` 提供 GET 接口（分别返回 HTML 与 PDF）。

## 四、前端实现说明（uni-app）

### 4.1 页面与路由

`HealthControl.uniapp/pages.json` 已注册如下就医辅助相关页面：

- 就医辅助首页：`pages/Front/MedicalHome`  
- 医院/医生列表：`pages/Front/HospitalDoctorList`  
- 医院/医生详情：`pages/Front/HospitalDoctorDetail`  
- 就医建议列表：`pages/Front/MedicalRecommendList`  
- 就医建议详情：`pages/Front/MedicalRecommendDetail`  
- 就诊准备报告（列表/生成）：`pages/Front/MedicalPrepareReport`  
- 报告导出记录列表：`pages/Front/MedicalExportRecordList`  
- 报告 HTML 预览页（web-view）：`pages/Front/MedicalReportWebView`

入口整合：

- `Index.vue`：首页增加“就医辅助”功能卡片，跳转 `/pages/Front/MedicalHome`；  
- `UserCenter.vue`：个人中心菜单中增加“就医辅助”入口，跳转 `/pages/Front/MedicalHome`；  
- `AudioResult.vue`：检测结果页增加“🏥 查看就医辅助”按钮，引导用户从异常检测结果快速进入就医路径；  
- `HealthRecord.vue`：健康档案页增加“就医辅助”卡片，引导用户从长期档案视角进入就医模块。

### 4.2 HTTP 封装与登录态

- 统一使用 `utils/http.js` 中的 `Post` 方法调用后端接口，自动携带 `Authorization: Bearer <token>`；  
- `store` 中的 `commonStore` 提供 `UserId/UserInfo/Token` 等字段，页面在调用前根据是否登录决定：
  - 未登录：部分页面只读提示（如就医首页显示“请先登录以查看个人数据”）；
  - 涉及用户数据的操作（生成报告、查看推荐详情、创建导出记录等）必须登录。

### 4.3 核心页面交互概要

- **MedicalHome**：
  - `onShow` 生命周期中调用 `/Front/Medical/Home` 拉取聚合数据；
  - 各卡片点击跳转到对应列表或详情页面；
  - 对于高风险用户，风险卡片文案会更强调“建议尽快线下就医/复查”。
- **HospitalDoctorList / Detail**：
  - 顶部搜索栏 + 地区、科室类型筛选条；
  - 列表支持下拉刷新与上拉加载；
  - 详情页提供电话拨打与预约链接跳转（含外链提示）。
- **MedicalRecommendList / Detail**：
  - 列表支持按“未查看/全部”切换；
  - 详情页 `onLoad` 时请求推荐详情并自动上报已查看；
  - 页面中内嵌医生卡片，支持一键拨号/跳转预约。
- **MedicalPrepareReport**：
  - 顶部展示生成报告表单（选择模板类型/内容范围）与“生成报告”按钮；
  - 下方展示历史报告列表；
  - 点击某条报告：
    - 若有 `HtmlDocUrl` → 跳 `MedicalReportWebView` 内嵌浏览；
    - 若有 `PdfDocUrl` → 可通过“下载 PDF”按钮调用 `uni.downloadFile + uni.openDocument`；
    - 打开/下载动作中，由前端调用 `/Front/Medical/ExportRecordCreate` 写入导出记录。
- **MedicalExportRecordList**：
  - 简单列表展示导出时间、格式、用途与文件链接；
  - 主要用于“行为可追溯”与用户回看历史导出动作。

## 五、小程序端调试与验收建议

### 5.1 最小验收路径（覆盖 MVP 要求）

1. 使用普通用户账号登录小程序。  
2. 在首页或个人中心点击“就医辅助”，进入 `MedicalHome`：  
   - 若已有健康档案/检测记录，确认风险概览、最近检测摘要展示正常；  
   - 若暂无数据，页面应展示友好的引导文案（去自查/去完善档案）。  
3. 从首页卡片进入“医院/医生库”，验证：  
   - 列表分页与筛选可用；  
   - 点击电话可正常拨号；  
   - 点击预约链接有二次确认提示并能打开/复制。  
4. 手动在数据库写入或通过后续规则生成一条 `TUserMedicalRecommend`：  
   - 在 `MedicalRecommendList` 中能看到列表项；  
   - 进入 `MedicalRecommendDetail` 时自动标记为已查看。  
5. 在 `MedicalPrepareReport` 页面点击“生成报告”：  
   - 前端提示生成中，随后列表出现一条新报告，状态为“已完成”；  
   - 点击“查看 HTML 报告” → 打开 `MedicalReportWebView`，可看到包含风险等级、检测/症状摘要与提示文案的完整 HTML 页；  
   - 点击“下载 PDF” → 触发下载并可在系统中打开；  
   - 导出/查看过程中，后台 `TScreenReportExportRecord` 有对应记录，报告表 `ViewCount/DownloadCount` 有更新。  
6. 在 `MedicalExportRecordList` 中看到刚刚的导出记录，验证导出格式/用途字段展示正确。

### 5.2 异常与兜底场景

- 无医院数据：  
  - 列表应展示“暂无可用医院/医生信息，请稍后重试或联系管理员”类文案，而不是空白页。  
- 推荐/报告越权访问：  
  - 使用非本人的用户 ID 访问推荐/报告详情接口时，后端返回错误码与友好错误信息；前端展示统一错误提醒。  
- 报告生成失败：  
  - `GenerateStatus=3` 时，前端在报告列表/详情中展示失败原因与“重试生成”按钮（若已实现 `PrepareReportRetry`）。  
- HTML/PDF 资源异常：  
  - 若状态为“已完成”但 `HtmlDocUrl/PdfDocUrl` 为空，前端提示“报告资源异常，请尝试重新生成或联系管理员”。

## 六、后续可优化点（基于当前实现）

1. **就医推荐触发与策略**  
   - 结合 `TAudioScreenRecord/TPersonalLaryngealHealthRecord/TSymptomLog` 等表，在检测完成或风险升级时自动生成推荐记录，并通过 `MessageNotice` 推送。  
2. **就诊计划与提醒**  
   - 基于建议文档中的 `TUserMedicalVisitPlan`，实现“我计划去就诊”与“就诊前 1 天提醒”等能力。  
3. **医生收藏与个人就医资源库**  
   - 引入 `TUserMedicalFavorite`，在医院/医生列表与详情页支持“收藏/取消收藏”，在个人中心增加“我的医生/医院”列表。  
4. **报告模板与内容扩展**  
   - 在当前 HTML 报告的基础上，进一步补充用户基本信息、问卷结果摘要、症状趋势统计、饮食记录按“友好/忌口”归类展示等内容，使其更接近真实门诊资料。  

当前版本已经满足就医辅助模块的 **MVP 闭环要求**，可以在真实用户路径中完成：  
“自查 → 发现风险 → 打开就医辅助 → 浏览医院/医生 → 查看就医建议 → 生成与导出就诊准备报告 → 记录导出行为” 的完整流程。
