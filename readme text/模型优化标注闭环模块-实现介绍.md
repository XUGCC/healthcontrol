## 模型优化标注闭环模块 - 实现介绍文档

## 一、模块概述

模型优化标注闭环模块是居家喉部健康自查小程序的核心数据闭环模块，旨在将"自查结果"与"真实确诊"连接起来，形成"**自查 → 就医 → 确诊 → 标注/授权 → 模型优化 → 更准自查**"的完整数据闭环，持续提升模型准确率，形成产品技术护城河。

当前版本已完成 **前台（UniApp）+ 后台（Spring Boot）+ 管理端（ElementUI）** 的闭环实现，满足需求文档《模型优化标注闭环模块-需求文档.md》中定义的所有功能需求（包括可选功能）。

## 二、主要实现功能概览

### 2.1 确诊标注表单页（ModelLabelForm）

- **页面**：`pages/Front/ModelLabelForm.vue`  
- **后端接口**：`POST /Front/ModelLabel/CreateOrEdit`（`FrontModelLabelController.CreateOrEdit` → `TModelOptimizeLabelService.FrontCreateOrEdit`）  
- **主要能力**：
  - **检测记录选择器**：
    - 自动关联：从检测结果页或就医建议页进入时，自动关联当前检测记录
    - 手动选择：提供下拉选择器，可选择历史检测记录（最近10条）
    - 显示格式：`检测时间 - 初筛结果 (置信度: XX%)`
  - **医院确诊结果输入**：
    - 文本输入框（多行，最多500字）
    - 快捷选择按钮：良性病变、恶性病变、正常/无异常、其他
    - 实时字数统计显示：`XX/500`
    - 标注类型自动判断：根据模型初筛结果和确诊结果关键词自动判断标注类型

    - 单选框：误报（模型判断为恶性，但实际为良性）/ 确诊（模型判断与实际情况一致）
    - 支持自动判断，用户可手动调整
  - **标注说明输入**：
    - 多行文本输入框（最多512字）
    - 实时字数统计显示：`XX/512`
    - 占位符提示："可补充说明确诊过程、检查方式、医生建议等信息（选填）"
  - **授权状态设置**：
    - 开关组件，默认根据隐私设置状态
    - 开启授权时显示确认提示对话框
    - 如果隐私设置未开启，提示引导用户前往隐私设置页
    - 授权状态与隐私设置 `DataAnonymousAuth` 联动
  - **表单提交**：
    - 必填项校验：检测记录、确诊结果、标注类型
    - 提交成功后显示成功提示，1.5秒后自动返回上一页
    - 支持创建新标注或更新已有标注（同一检测记录可多次标注）

## 三、后端实现说明（Spring Boot）

### 3.1 核心实体与 Mapper

- **模型优化标注表**：`TModelOptimizeLabel`  
  - 关键字段：`Id, UserId, DetectId, HospitalDiagnoseResult, LabelType, LabelDesc, AuthStatus, IsDelete, CreationTime, UpdateTime`  
  - `LabelType`：标注类型（false=误报，true=确诊）
  - `AuthStatus`：授权状态（false=未授权，true=已授权）
  - `IsDelete`：软删除标记
- **用户隐私设置表**：`TUserPrivacySetting`  
  - 关键字段：`UserId, DataAnonymousAuth`  
  - `DataAnonymousAuth`：数据匿名授权状态，与标注的 `AuthStatus` 联动
- **音频自查记录表**：`TAudioScreenRecord`  
  - 用于关联检测记录，获取初筛结果和置信度

### 3.2 前台业务 Service（`TModelOptimizeLabelService`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/TModelOptimizeLabelService.java`  
主要方法：

- `FrontCreateOrEdit(TModelOptimizeLabelDto)`  
  - 创建或更新标注记录
  - 校验用户登录状态（从 `BaseContext` 获取）
  - 校验检测记录归属（防止越权）
  - 如果 `AuthStatus = true`，同步更新隐私设置 `DataAnonymousAuth = true`
  - 支持同一检测记录多次标注（不覆盖旧记录）
- `FrontList(TModelOptimizeLabelPagedInput)`  
  - 获取用户标注列表
  - 自动过滤软删除记录（`IsDelete = false`）
  - 关联检测记录信息（`DetectDto`）
  - 按创建时间倒序排序
  - 支持分页（Page, Limit）
- `FrontGet(TModelOptimizeLabelPagedInput)`  
  - 获取单条标注详情
  - 校验标注记录归属（防止越权）
  - 关联检测记录详细信息
- `FrontUpdateAuthStatus(Integer id, Boolean authStatus)`  
  - 更新授权状态
  - 校验标注记录归属
  - 如果 `AuthStatus = true`，同步更新隐私设置
- `FrontDelete(Integer id)`  
  - 软删除标注记录
  - 校验标注记录归属
  - 设置 `IsDelete = true`
- `FrontGetDetectRecords(Integer userId)`  
  - 获取用户检测记录列表（用于选择关联）
  - 返回用户最近的检测记录（按创建时间倒序）
  - 自动过滤软删除记录

内部辅助方法：

- `checkAndUpdatePrivacySetting(Integer userId, Boolean authStatus)`  
  - 校验并更新隐私设置中的匿名授权状态
  - 如果隐私设置记录不存在，自动创建
  - 如果开启授权，更新 `DataAnonymousAuth = true`
- `validateDetectOwnership(Integer detectId, Integer userId)`  
  - 校验检测记录归属
  - 防止用户越权访问其他用户的检测记录

### 3.3 前台控制器（`FrontModelLabelController`）

位置：`HealthControl.springboot/src/main/java/com/example/web/controller/FrontModelLabelController.java`  
主要职责：

- 暴露 `/Front/ModelLabel/*` 前台 REST 接口：
  - `CreateOrEdit`（创建或更新标注）
  - `List`（获取用户标注列表）
  - `Get`（获取单条标注详情）
  - `UpdateAuth`（更新授权状态）
  - `Delete`（删除标注）
  - `GetDetectRecords`（获取用户检测记录列表）
- 统一封装响应为 `{ Code, Success, Msg, Data }`
- 所有接口自动从 `BaseContext` 获取当前登录用户ID，确保数据安全

### 3.4 管理端增强（`TModelOptimizeLabelList.vue`）

位置：`HealthControl.elementui/src/views/Admin/TModelOptimizeLabelList.vue`  
主要能力：

- **筛选功能**：
  - 授权状态筛选：全部/已授权/未授权
  - 标注类型筛选：全部/误报/确诊
  - 其他筛选：医院确诊结果、标注说明、用户、检测记录、时间范围
- **统计信息卡片**：
  - 总标注数
  - 已授权数（绿色显示）
  - 未授权数（灰色显示）
  - 误报/确诊比例
- **审核功能**（简化版本）：
  - 审核状态列：基于 `UpdateTime` 判断（如果 `UpdateTime` 不等于 `CreationTime`，视为已审核）
  - 标记已审核/取消审核按钮
  - 审核备注功能（代码已预留，需要数据库字段支持）
- **导出功能**：
  - 导出已授权标注按钮：仅导出 `AuthStatus = true` 的标注记录
  - 导出格式：Excel/CSV
  - 导出字段：标注ID、用户ID、检测ID、初筛结果、初筛置信度、医院确诊结果、标注类型、标注说明、创建时间

### 4.4 标注类型自动判断逻辑

标注类型自动判断逻辑在 `ModelLabelForm.vue` 的 `autoJudgeLabelType` 方法中实现：

```javascript
const autoJudgeLabelType = () => {
  if (!selectedDetect.value || !formData.value.HospitalDiagnoseResult) return
  
  const detectResult = selectedDetect.value.PrimaryScreenResult // 0=良性, 1=恶性倾向
  const diagnoseResult = formData.value.HospitalDiagnoseResult.toLowerCase()
  
  // 判断确诊结果类型
  const isBenignDiagnose = diagnoseResult.includes('良性') || 
                          diagnoseResult.includes('正常') || 
                          diagnoseResult.includes('无异常') ||
                          diagnoseResult.includes('小结') ||
                          diagnoseResult.includes('息肉')
  
  const isMalignantDiagnose = diagnoseResult.includes('恶性') || 
                             diagnoseResult.includes('癌') ||
                             diagnoseResult.includes('肿瘤')
  
  // 如果模型判断为恶性(1)，但确诊为良性，则为误报(false)
  // 如果模型判断与确诊结果一致，则为确诊(true)
  if (detectResult === 1 && isBenignDiagnose) {
    formData.value.LabelType = false // 误报
  } else if ((detectResult === 1 && isMalignantDiagnose) || 
             (detectResult === 0 && isBenignDiagnose)) {
    formData.value.LabelType = true // 确诊
  }
  // 其他情况不自动判断，让用户手动选择
}
```

触发时机：
- 选择检测记录后
- 选择快捷结果后
- 输入确诊结果后失去焦点（`@blur`）

## 五、数据库表设计

### 5.1 模型优化标注表（tmodeloptimizelabel）

**表结构**（基于现有表，无需修改）：

| 字段名 | 类型 | 说明 | 必填 | 备注 |
|--------|------|------|------|------|
| Id | Integer | 主键ID | 是 | 自增 |
| CreationTime | LocalDateTime | 创建时间 | 是 | 自动填充 |
| UpdateTime | LocalDateTime | 授权状态更新时间 | 否 | |
| UserId | Integer | 关联用户ID | 是 | 外键，关联AppUser表 |
| DetectId | Integer | 关联音频自查记录ID | 是 | 外键，关联TAudioScreenRecord表 |
| HospitalDiagnoseResult | String(255) | 医院确诊结果 | 是 | 用户输入的确诊结果 |
| LabelType | Boolean | 标注类型 | 是 | false=误报，true=确诊 |
| LabelDesc | String(512) | 标注说明 | 否 | 补充说明信息 |
| AuthStatus | Boolean | 授权状态 | 是 | false=未授权，true=已授权 |
| IsDelete | Boolean | 软删除标记 | 是 | false=未删除，true=已删除 |

**新增索引**（通过 `模型优化标注闭环模块-数据库修改.sql` 执行）：

- `idx_user_auth_time`：(`UserId`, `AuthStatus`, `CreationTime`) - 用于列表查询和授权筛选
- `idx_detect_delete`：(`DetectId`, `IsDelete`) - 用于查询检测记录的标注

**业务约束**：
1. 同一检测记录可以有多条标注记录（支持复查更新）
2. 授权状态可以随时修改
3. 软删除的记录不显示在用户列表中，但管理员可查看

### 5.2 用户隐私设置表（tuserprivacysetting）

**表结构**（基于现有表，无需修改）：

| 字段名 | 类型 | 说明 | 必填 | 备注 |
|--------|------|------|------|------|
| Id | Integer | 主键ID | 是 | 自增 |
| UserId | Integer | 关联用户ID | 是 | 外键，关联AppUser表 |
| DataAnonymousAuth | Boolean | 数据匿名授权状态 | 是 | false=未授权，true=已授权 |
| IsDelete | Boolean | 软删除标记 | 是 | false=未删除，true=已删除 |

**业务约束**：
1. 每个用户只有一条有效隐私设置记录（`IsDelete = false`）
2. `DataAnonymousAuth` 与标注的 `AuthStatus` 联动
3. 标注时如果 `AuthStatus = true`，自动同步更新 `DataAnonymousAuth = true`

### 5.3 审核功能字段（可选）

如需完整审核功能（包括审核备注），可执行数据库脚本中的审核字段添加语句：

```sql
ALTER TABLE `tmodeloptimizelabel`
  ADD COLUMN `ReviewStatus` TINYINT(1) NOT NULL DEFAULT 0 COMMENT '审核状态：0=待审核,1=已审核,2=需复核' AFTER `AuthStatus`,
  ADD COLUMN `ReviewRemark` VARCHAR(512) NULL COMMENT '审核备注' AFTER `ReviewStatus`,
  ADD COLUMN `ReviewTime` DATETIME NULL COMMENT '审核时间' AFTER `ReviewRemark`,
  ADD COLUMN `ReviewerId` INT(11) NULL COMMENT '审核人ID' AFTER `ReviewTime`;
```

### 6.2 异常与兜底场景

- **无标注数据**：
  - 列表应展示"暂无标注记录，快去提交您的确诊结果吧"，而不是空白页
  - 统计卡片显示"0"
- **标注越权访问**：
  - 使用非本人的用户ID访问标注详情接口时，后端返回错误码

  - 用户可以选择前往设置或取消授权
- **标注类型自动判断失败**：
  - 如果自动判断无法确定标注类型，用户可手动选择
  - 不会强制要求自动判断结果

### 7.2 隐私设置联动逻辑

```java
/**
 * 校验并更新隐私设置中的匿名授权状态
 */
private void checkAndUpdatePrivacySetting(Integer userId, Boolean authStatus) {
    if (!authStatus) return; // 关闭授权时不强制更新隐私设置
    
    TUserPrivacySetting privacySetting = privacySettingMapper.selectOne(
        Wrappers.<TUserPrivacySetting>lambdaQuery()
            .eq(TUserPrivacySetting::getUserId, userId)
            .eq(TUserPrivacySetting::getIsDelete, false)
    );
    
    if (privacySetting == null) {
        // 创建隐私设置记录
        privacySetting = new TUserPrivacySetting();
        privacySetting.setUserId(userId);
        privacySetting.setCreationTime(LocalDateTime.now());
        privacySetting.setDataAnonymousAuth(true);
        privacySetting.setLocalStorageStatus(false);
        privacySetting.setPrivacyAgreeStatus(false);
        privacySetting.setIsDelete(false);
        privacySettingMapper.insert(privacySetting);
    } else if (!privacySetting.getDataAnonymousAuth()) {
        // 更新隐私设置
        privacySetting.setDataAnonymousAuth(true);
        privacySettingMapper.updateById(privacySetting);
    }
}
```

### 7.3 标注类型自动判断逻辑

前端自动判断逻辑：

```javascript
// 判断确诊结果类型
const isBenignDiagnose = diagnoseResult.includes('良性') || 
                        diagnoseResult.includes('正常') || 
                        diagnoseResult.includes('无异常') ||
                        diagnoseResult.includes('小结') ||
                        diagnoseResult.includes('息肉')

const isMalignantDiagnose = diagnoseResult.includes('恶性') || 
                           diagnoseResult.includes('癌') ||
                           diagnoseResult.includes('肿瘤')

// 如果模型判断为恶性(1)，但确诊为良性，则为误报(false)
// 如果模型判断与确诊结果一致，则为确诊(true)
if (detectResult === 1 && isBenignDiagnose) {
  formData.value.LabelType = false // 误报
} else if ((detectResult === 1 && isMalignantDiagnose) || 
           (detectResult === 0 && isBenignDiagnose)) {
  formData.value.LabelType = true // 确诊
}
```

// 判断是否已审核（基于UpdateTime判断）
const getIsReviewed = (row) => {
  if (!row.UpdateTime || !row.CreationTime) return false;
  // 如果UpdateTime和CreationTime不同，视为已审核（简化判断）
  return row.UpdateTime !== row.CreationTime;
};

// 标记为已审核/取消审核
const MarkAsReviewed = async (row) => {
  const isReviewed = getIsReviewed(row);
  const newStatus = !isReviewed;
  
  // 更新标注记录，通过修改UpdateTime来标记审核状态
  const updateData = {
    ...row,
    UpdateTime: newStatus ? new Date().toISOString().replace('T', ' ').substring(0, 19) : row.CreationTime
  };
  
  const { Success } = await Post(`/TModelOptimizeLabel/CreateOrEdit`, updateData);
  if (Success) {
    ElMessage.success(newStatus ? '已标记为已审核' : '已取消审核标记');
    PaginationTableId.value.Reload(searchForm);
