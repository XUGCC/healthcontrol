# 模型优化标注闭环模块 - 详细需求文档

## 一、模块概述

### 1.1 模块定位
模型优化标注闭环模块是居家喉部健康自查小程序的核心数据闭环模块，旨在将"自查结果"与"真实确诊"连接起来，形成"**自查 → 就医 → 确诊 → 标注/授权 → 模型优化 → 更准自查**"的完整数据闭环，持续提升模型准确率，形成产品技术护城河。

### 1.2 核心价值
- **数据闭环**：连接自查结果与真实确诊，形成可训练的高质量数据集
- **模型迭代**：基于真实临床对照数据，持续优化 Densenet 模型准确率
- **合规保障**：完善的用户授权机制，符合数据隐私法规要求
- **产品护城河**：积累真实临床数据，形成技术壁垒和竞争优势
- **用户价值**：帮助用户记录和追踪就医确诊结果，形成个人医疗档案

### 1.3 与其他模块的关系
- **依赖模块**：
  - 用户基础模块（用户信息、隐私设置）
  - 音频自查分析模块（检测记录，用于关联标注）
  - 就医辅助模块（就医建议，用于引导用户就医）
- **支撑模块**：为模型训练和优化提供高质量标注数据
- **数据来源**：
  - `tmodeloptimizelabel`：模型优化标注表
  - `tuserprivacysetting`：用户隐私设置表（授权状态）
  - `taudioscreenrecord`：音频自查记录表（关联检测）

## 二、功能需求详细说明

### 2.1 确诊标注入口（多入口设计）

#### 2.1.1 功能描述
在用户可能产生确诊结果的多个场景下，提供便捷的标注入口，引导用户提交确诊信息。

#### 2.1.2 入口位置

**入口1：检测结果页（AudioResult.vue）**
- **位置**：检测结果卡片下方，操作按钮区域
- **按钮文案**："我已去医院检查" / "我有确诊结果"
- **显示条件**：
  - 用户已登录
  - 当前检测记录存在（`DetectId` 有效）
  - 该检测记录尚未有标注记录（可选，避免重复标注）
- **交互**：点击后跳转到标注表单页，自动关联当前检测记录

**入口2：个人中心（UserCenter.vue）**
- **位置**：功能菜单区域，新增"确诊标注与数据授权"菜单项
- **图标**：使用医疗相关图标（如🏥或📋）
- **角标**：如有未授权标注，显示角标提示
- **交互**：点击后跳转到标注记录列表页

**入口3：就医建议详情页（MedicalRecommendDetail.vue）**
- **位置**：推荐医生卡片下方，操作按钮区域
- **按钮文案**："我已就诊" / "提交确诊结果"
- **显示条件**：
  - 用户已登录
  - 存在关联的检测记录（从 `RecommendId` 关联到 `DetectId`）
- **交互**：点击后跳转到标注表单页，自动关联相关检测记录

**入口4：健康档案页（HealthRecord.vue）（可选）**
- **位置**：档案详情区域，新增"确诊标注"入口
- **交互**：点击后跳转到标注记录列表页，可查看历史标注

#### 2.1.3 业务规则
1. 同一检测记录可以多次标注（如用户复查后更新确诊结果）
2. 标注入口仅在用户登录状态下显示
3. 入口文案可根据场景调整，但核心功能一致

### 2.2 确诊标注表单页（ModelLabelForm.vue）

#### 2.2.1 功能描述
用户提交医院确诊结果的表单页面，支持关联检测记录、输入确诊信息、选择标注类型、设置授权状态。

#### 2.2.2 页面结构

**顶部导航栏**：
- 标题："确诊标注"
- 返回按钮：返回上一页

**表单内容区域**：

1. **关联检测记录选择**
   - **自动关联**：如果从检测结果页或就医建议页进入，自动关联当前检测记录
   - **手动选择**：提供下拉选择器，可选择历史检测记录
   - **检测记录展示**：
     - 检测时间
     - 初筛结果（良性/恶性）
     - 置信度
     - 检测ID（可选显示）
   - **业务规则**：
     - 必须选择一个检测记录
     - 支持选择历史检测记录（不限于最近一次）

2. **医院确诊结果输入**
   - **输入方式**：
     - 文本输入框（多行，最多500字）
     - 常见结果快捷选择（可选）：
       - 良性病变（声带小结、声带息肉、慢性咽炎等）
       - 恶性病变（喉癌、声带癌等）
       - 正常/无异常
       - 其他（手动输入）
   - **输入提示**：
     - 占位符："请输入医院确诊结果，如：声带小结、声带息肉、喉癌等"
     - 提示文案："请如实填写医院检查的确诊结果，这将有助于模型优化"
   - **业务规则**：
     - 确诊结果为必填项
     - 支持输入医院诊断报告的关键信息

3. **标注类型选择**
   - **选项**：
     - `false`（0）：误报（模型判断为恶性，但实际为良性）
     - `true`（1）：确诊（模型判断与实际情况一致）
     - 其他（可选扩展）
   - **显示方式**：单选框或开关
   - **默认值**：根据模型初筛结果和确诊结果自动判断（可选）
   - **业务规则**：
     - 标注类型为必选项
     - 如果用户选择"确诊"，系统可自动判断是否与模型结果一致

4. **标注说明（可选）**
   - **输入方式**：多行文本输入框
   - **字数限制**：最多512字
   - **用途**：补充说明、特殊情况说明等
   - **占位符**："可补充说明确诊过程、检查方式、医生建议等信息（选填）"

5. **授权状态设置**
   - **显示方式**：开关组件
   - **开关状态**：
     - 开启（`AuthStatus = true`）：授权匿名数据用于模型优化
     - 关闭（`AuthStatus = false`）：不授权
   - **默认值**：
     - 如果用户隐私设置中 `DataAnonymousAuth = true`，默认开启
     - 否则默认关闭
   - **提示文案**：
     - 开启时："您的匿名数据将用于模型优化，帮助提升检测准确率"
     - 关闭时："您的数据将仅用于个人记录，不参与模型优化"
   - **隐私设置联动**：
     - 如果用户隐私设置中 `DataAnonymousAuth = false`，提示用户："您尚未开启数据匿名授权，是否前往隐私设置开启？"
     - 提供"前往设置"按钮，跳转到隐私设置页

6. **提交按钮**
   - **按钮文案**："提交标注"
   - **按钮样式**：主色调按钮，宽度填充
   - **提交前校验**：
     - 检测记录必选
     - 确诊结果必填
     - 标注类型必选
   - **提交后反馈**：
     - 显示"提交成功"提示
     - 1秒后自动返回上一页或跳转到标注记录列表页

#### 2.2.3 交互功能

1. **检测记录选择**
   - 点击选择器，弹出检测记录列表（最近10条）
   - 列表显示：检测时间、初筛结果、置信度
   - 选择后自动填充到表单

2. **确诊结果输入**
   - 支持文本输入和快捷选择
   - 快捷选择后自动填充到输入框，用户可继续编辑
   - 实时显示字数统计（可选）

3. **授权状态切换**
   - 切换开关时，显示确认提示（可选）
   - 如果隐私设置未开启，提示引导开启

4. **表单提交**
   - 提交前进行必填项校验
   - 提交中显示加载状态
   - 提交成功后显示成功提示
   - 提交失败显示错误提示

#### 2.2.4 业务规则

1. **数据校验**：
   - 检测记录ID必须有效
   - 确诊结果不能为空
   - 标注类型必须选择

2. **授权状态联动**：
   - 如果用户隐私设置中 `DataAnonymousAuth = false`，标注时提示用户开启
   - 标注提交时，如果 `AuthStatus = true`，同步更新隐私设置 `DataAnonymousAuth = true`（可选）

3. **重复标注**：
   - 同一检测记录可以多次标注（如复查后更新结果）
   - 每次标注创建新记录，不覆盖旧记录
   - 列表页显示最新一条标注

#### 2.5.2 授权状态校验逻辑

**Service层实现**（`TModelOptimizeLabelServiceImpl`）：

```java
/**
 * 创建或更新标注（前台接口）
 */
public TModelOptimizeLabelDto CreateOrEdit(TModelOptimizeLabelDto input) {
    // 1. 校验用户登录状态（从BaseContext获取）
    Integer currentUserId = BaseContext.getCurrentId();
    if (currentUserId == null || !currentUserId.equals(input.getUserId())) {
        throw new CustomException("用户未登录或无权操作");
    }
    
    // 2. 校验检测记录归属
    TAudioScreenRecord detect = audioScreenRecordMapper.selectById(input.getDetectId());
    if (detect == null || !detect.getUserId().equals(input.getUserId())) {
        throw new CustomException("检测记录不存在或无权访问");
    }
    
    // 3. 如果授权状态为true，校验隐私设置
    if (input.getAuthStatus() != null && input.getAuthStatus()) {
        TUserPrivacySetting privacySetting = privacySettingMapper.selectOne(
            Wrappers.<TUserPrivacySetting>lambdaQuery()
                .eq(TUserPrivacySetting::getUserId, input.getUserId())
                .eq(TUserPrivacySetting::getIsDelete, false)
        );
        
        if (privacySetting == null || 
            (privacySetting.getDataAnonymousAuth() != null && !privacySetting.getDataAnonymousAuth())) {
            // 提示用户开启隐私设置中的匿名授权
            // 可选：自动开启隐私设置
            // privacySetting.setDataAnonymousAuth(true);
            // privacySettingMapper.updateById(privacySetting);
        }
    }
    
    // 4. 创建或更新标注记录
    TModelOptimizeLabel entity = input.MapToEntity();
    if (input.getId() == null) {
        entity.setCreationTime(LocalDateTime.now());
        modelOptimizeLabelMapper.insert(entity);
    } else {
        entity.setUpdateTime(LocalDateTime.now());
        modelOptimizeLabelMapper.updateById(entity);
    }
    
    // 5. 如果授权状态为true，同步更新隐私设置（可选）
    if (input.getAuthStatus() != null && input.getAuthStatus()) {
        updatePrivacySettingAuth(input.getUserId(), true);
    }
    
    return entity.MapToDto();
}

/**
 * 更新隐私设置中的匿名授权状态
 */
private void updatePrivacySettingAuth(Integer userId, Boolean authStatus) {
    TUserPrivacySetting privacySetting = privacySettingMapper.selectOne(
        Wrappers.<TUserPrivacySetting>lambdaQuery()
            .eq(TUserPrivacySetting::getUserId, userId)
            .eq(TUserPrivacySetting::getIsDelete, false)
    );
    
    if (privacySetting == null) {
        // 创建隐私设置记录
        privacySetting = new TUserPrivacySetting();
        privacySetting.setUserId(userId);
        privacySetting.setCreationTime(LocalDateTime.now());
        privacySetting.setDataAnonymousAuth(authStatus);
        privacySetting.setIsDelete(false);
        privacySettingMapper.insert(privacySetting);
    } else {
        // 更新隐私设置
        privacySetting.setDataAnonymousAuth(authStatus);
        privacySettingMapper.updateById(privacySetting);
    }
}
```

```java
@Override
public ResultDto<TModelOptimizeLabelDto> CreateOrEditForFront(TModelOptimizeLabelDto input) {
    // 1. 校验用户登录状态
    Integer currentUserId = BaseContext.getCurrentId();
    if (currentUserId == null || !currentUserId.equals(input.getUserId())) {
        throw new CustomException("用户未登录或无权操作");
    }
    
    // 2. 校验检测记录归属
    TAudioScreenRecord detect = audioScreenRecordMapper.selectById(input.getDetectId());
    if (detect == null || !detect.getUserId().equals(input.getUserId())) {
        throw new CustomException("检测记录不存在或无权访问");
    }
    
    // 3. 授权状态校验和隐私设置联动
    if (input.getAuthStatus() != null && input.getAuthStatus()) {
        checkAndUpdatePrivacySetting(input.getUserId(), true);
    }
    
    // 4. 创建或更新标注
    TModelOptimizeLabel entity = input.MapToEntity();
    if (input.getId() == null) {
        entity.setCreationTime(LocalDateTime.now());
        modelOptimizeLabelMapper.insert(entity);
    } else {
        entity.setUpdateTime(LocalDateTime.now());
        modelOptimizeLabelMapper.updateById(entity);
    }
    
    return ResultDto.Success(entity.MapToDto());
}

public PagedResult<TAudioScreenRecordDto> GetUserDetectListForFront(TModelOptimizeLabelPagedInput input) {
    Integer currentUserId = BaseContext.getCurrentId();
    if (currentUserId == null) {
        throw new CustomException("用户未登录");
    }
    
    // 查询用户最近的检测记录
    LambdaQueryWrapper<TAudioScreenRecord> queryWrapper = Wrappers
        .<TAudioScreenRecord>lambdaQuery()
        .eq(TAudioScreenRecord::getUserId, currentUserId)
        .eq(TAudioScreenRecord::getIsDelete, false)
        .orderByDesc(TAudioScreenRecord::getCreationTime);
    
    Page<TAudioScreenRecord> page = new Page<>(input.getPage(), input.getLimit());
    IPage<TAudioScreenRecord> pageResult = audioScreenRecordMapper.selectPage(page, queryWrapper);
    
    List<TAudioScreenRecordDto> items = pageResult.getRecords().stream()
        .map(entity -> {
            try {
                return entity.MapToDto();
            } catch (Exception e) {
                return null;
            }
        })
        .filter(Objects::nonNull)
        .collect(Collectors.toList());
    
    PagedResult<TAudioScreenRecordDto> result = new PagedResult<>();
    result.setItems(items);
    result.setTotalCount((int) pageResult.getTotal());
    return result;
}

private void checkAndUpdatePrivacySetting(Integer userId, Boolean authStatus) {
    TUserPrivacySetting privacySetting = privacySettingMapper.selectOne(
        Wrappers.<TUserPrivacySetting>lambdaQuery()
            .eq(TUserPrivacySetting::getUserId, userId)
            .eq(TUserPrivacySetting::getIsDelete, false)
    );
    
    if (privacySetting == null) {
        // 创建隐私设置记录
        privacySetting = new TUserPrivacySetting();
        privacySetting.setUserId(userId);
        privacySetting.setCreationTime(LocalDateTime.now());
        privacySetting.setDataAnonymousAuth(authStatus);
        privacySetting.setLocalStorageStatus(false);
        privacySetting.setPrivacyAgreeStatus(false);
        privacySetting.setIsDelete(false);
        privacySettingMapper.insert(privacySetting);
    } else if (authStatus && 
               (privacySetting.getDataAnonymousAuth() == null || !privacySetting.getDataAnonymousAuth())) {
        // 更新隐私设置
        privacySetting.setDataAnonymousAuth(true);
        privacySettingMapper.updateById(privacySetting);
    }
}
```

### 6.1 标注提交流程

```
用户进入标注表单页
    ↓
选择关联检测记录（自动关联或手动选择）
    ↓
输入医院确诊结果
    ↓
选择标注类型（误报/确诊）
    ↓
填写标注说明（可选）
    ↓
设置授权状态
    ├─ 开启授权 → 校验隐私设置
    │   ├─ 隐私设置已开启 → 直接提交
    │   └─ 隐私设置未开启 → 提示用户开启
    └─ 关闭授权 → 直接提交
    ↓
提交标注
    ↓
后端校验（用户登录、检测记录归属、授权状态）
    ↓
创建标注记录
    ↓
如果授权开启，同步更新隐私设置
    ↓
返回成功，前端跳转
```

```
后端校验权限
```

### 12.5 隐私设置联动说明

**联动规则**：
1. 用户在标注时开启授权（`AuthStatus = true`），系统自动检查隐私设置
2. 如果隐私设置中 `DataAnonymousAuth = false`，提示用户开启
3. 用户确认后，系统自动更新隐私设置 `DataAnonymousAuth = true`
4. 如果隐私设置记录不存在，系统自动创建

**数据一致性**：
- 标注的 `AuthStatus` 与隐私设置的 `DataAnonymousAuth` 应保持一致
- 建议在标注提交和授权状态修改时，同步更新隐私设置
- 管理员导出数据时，仅导出 `AuthStatus = true` 的记录
