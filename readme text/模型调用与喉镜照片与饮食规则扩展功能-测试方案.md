## 二、模型检测服务透明度与稳定性反馈测试

### 2.1 小程序端：音频结果页“模型服务状态”卡片（AudioResult.vue）

**用例 2.1.1 有完整成功日志的检测**
- 步骤：
  - 使用用户 A 完成一次音频自查，确保后端为该 `DetectId` 记录多条成功的 `TModelInterfaceCallLog`。
  - 自查完成后进入 `AudioResult.vue` 结果页。
- 预期：
  - 在检测结果下方出现“模型服务状态”卡片。
  - 概览信息包括：总调用次数、成功次数、失败次数、平均耗时（或区间）等。
  - 状态文案为“本次检测模型服务运行顺畅”或等价“正常”文案。
  - 不出现报错或空白区域。

**用例 2.1.2 存在失败重试但最终成功的检测**
- 步骤：
  - 人为构造一条检测：同一 `DetectId` 下既有失败日志，又有最终成功日志，并在服务聚合结果中标记为“WARNING/有兜底”。
  - 打开对应检测的结果页。
- 预期：
  - “模型服务状态”卡片状态为“轻微异常/失败重试”等提示，文案说明“过程中出现异常，系统已自动重试/兜底处理”。
  - 详细列表中可看到失败和成功的调用记录（按 `CallLink` 分组展示）。

**用例 2.1.3 所有调用失败的检测**
- 步骤：
  - 构造一个 `DetectId`，对应的 `TModelInterfaceCallLog` 全部为失败。
  - 打开结果页（前端整体检测流程本身可能已有兜底提示）。
- 预期：
  - “模型服务状态”卡片显示“本次模型服务出现异常，请稍后重试”或类似错误提示。
  - 调用明细中所有环节状态为失败，展示错误码和简短错误信息。

**用例 2.1.4 无日志记录的检测**
- 步骤：
  - 构造一个旧数据或特殊流程的检测记录，数据库中没有对应 `TModelInterfaceCallLog`。
  - 打开结果页。
- 预期：
  - 不出现异常报错。
  - “模型服务状态”模块可隐藏或显示“暂无内部日志记录”提示。

**用例 2.1.5 折叠面板的“详细调用信息”**
- 步骤：
  - 在含有多条日志的检测结果页中，展开“查看详细调用信息”。
- 预期：
  - 按调用环节（CallLinkName）分组展示每条调用：接口类型、耗时、状态、错误码/摘要。
  - 下方安全说明文案存在：“仅用于技术透明展示，不代表医疗结论”等。

- 预期：
  - 结果列表仅包含 `ModelVersion` 或 `ServiceName` 字段中包含对应关键字的日志。

**用例 2.2.4 导出功能**
- 步骤：
  - 在当前筛选条件下，点击“导出”按钮。
- 预期：
  - 成功下载 Excel/导出文件。
  - 文件内容与当前筛选结果一致（记录数、字段）。

### 4.2 后端规则命中与 HitCount 计数（FrontDietService）

**用例 4.2.1 命中规则的 HitCount 增长**
- 步骤：

  - 推荐解释/场景卡片优先展示 `DisplayWeight` 高的规则；在权重相同的情况下，再按 `RecommendPriority` 排序。

## 五、安全与权限相关测试

**用例 5.1 模型调用日志的用户隔离（前台）**
- 步骤：
  - 用户 A 与用户 B 各自拥有不同的 `DetectId`。
  - 用户 A 尝试在小程序中通过某种方式访问 B 的 `DetectId`（如手动拼接参数）。
- 预期：
  - 后端在 `FrontModelService.callLogSummary` 中校验 `UserId`，对非本人记录返回无权限错误，前端给出友好提示。

**用例 5.2 喉镜照片的用户隔离**
- 步骤：
  - 用户 B 登录小程序，尝试访问用户 A 的喉镜照片记录 ID（通过 URL 篡改等方式）。
- 预期：
  - 后端接口根据 `UserId` 校验，只返回当前登录用户的数据。
  - 不会泄露他人喉镜照片 URL 或描述信息。

**用例 5.3 管理端操作的审慎性**
- 步骤：
  - 管理员在模型日志与饮食规则列表中进行删除/编辑操作。
- 预期：
  - 所有删除/保存操作都有二次确认或合理提示。
  - 日志记录关键操作（如规则配置变更），便于后续审计（如果项目已接日志系统）。
