## 消息通知中心模块（通知中心 + 行为闭环）- 实现介绍文档

## 一、模块概述

消息通知中心模块围绕"**提醒配置页升级为通知中心 + 行为闭环**"这一核心场景，为小程序用户提供：

- 统一的通知管理入口，支持筛选、搜索、分组展示；  
- 通知点击后能正确跳转到对应业务页面，形成完整的用户行为链路；  
- 重要/置顶标记功能，方便用户保留关键提醒；  
- 未读通知角标联动，提升通知触达效率；  
- "今天不再提醒"功能，提升用户体验；  
- 基于风险等级的个性化推送策略。

当前版本已完成 **前台（UniApp）+ 后台（Spring Boot）** 的闭环实现，满足需求文档《消息通知中心模块-需求文档.md》中定义的所有功能需求（包括可选功能）。

## 二、主要实现功能概览

### 2.1 通知列表页（ReminderMessageList）

- **页面**：`pages/Front/ReminderMessageList.vue`  
- **后端接口**：`POST /MessageNotice/List`（`MessageNoticeController.List` → `MessageNoticeService.List`）  
- **主要能力**：
  - **顶部统计卡片**：
    - 展示最近30天收到提醒总数、已阅读数量、未阅读数量
    - 统计信息从后端接口返回，支持时间范围配置
  - **搜索与日期筛选**：
    - 搜索框：支持按标题/内容关键字搜索
    - 日期筛选：全部/近7天/近30天（自定义日期功能预留）
  - **筛选标签栏**：
    - 全部、未读、自查提醒、护嗓提醒、随访提醒
    - 点击标签立即刷新列表，重置分页
  - **通知卡片列表（按日期分组）**：
    - 按"今天"、"近7天"、"更早"分组展示
    - 每个通知卡片显示：
      - 标题（支持重要标记橙色标签）
      - 内容（最多2行，超出省略）
      - 类型标签（自查=绿色，护嗓=蓝色，随访=橙色）
      - 未读标识（红色圆点）
      - 置顶图标（📌）
      - 发送时间（优先显示ActualSendTime）
    - 长按通知卡片弹出操作菜单：标记为重要/取消重要、置顶/取消置顶、删除
  - **底部操作栏**：
    - 全部标记为已读：批量标记当前筛选结果中的所有未读通知为已读
    - 清理已读：批量软删除所有已读通知
  - **通知点击跳转**：
    - 自动标记为已读
    - 根据 `ActionType` 和 `ActionParam` 跳转到对应页面：
      - `GOTO_AUDIO_DETECT` → 音频自查页
      - `GOTO_HEALTH_RECORD` → 健康档案页
      - `GOTO_SYMPTOM_FORM` → 症状记录表单页
      - `GOTO_DIET_HOME` → 饮食首页
      - `GOTO_MEDICAL_HOME` → 就医辅助首页
      - `GOTO_MEDICAL_RECOMMEND_DETAIL` → 就医建议详情页
      - `GOTO_AUDIO_RESULT` → 检测结果详情页
      - 无 `ActionType` 或未知类型 → 跳转到通知详情页
  - **分页加载**：
    - 滚动到底部自动加载下一页
    - 每页20条记录
    - 加载中显示"加载中..."提示

### 2.2 通知详情页（ReminderMessageDetail）

- **页面**：`pages/Front/ReminderMessageDetail.vue`  
- **后端接口**：`POST /MessageNotice/Get`  
- **核心实现**：
  - 展示完整标题、时间、类型、正文全文
  - 显示重要/置顶状态
  - 自动标记为已读（打开详情时）
  - 操作按钮：
    - 标记为重要/取消重要
    - 置顶/取消置顶
    - 删除
    - 跳转到相关页面（如有ActionType）
  - 页面加载时校验参数，无ID时提示错误并返回

### 2.3 提醒设置页增强（HealthReminder）

- **页面**：`pages/Front/HealthReminder.vue`  
- **后端接口**：
  - `POST /TPersonalHealthRemind/CreateOrEdit`（保存设置）
  - `POST /TPersonalHealthRemind/SnoozeToday`（今天不再提醒）
- **主要能力**：
  - **"今天不再提醒"功能**：
    - 在护嗓提醒卡片中提供"今天不再提醒"按钮
    - 点击后调用后端接口，更新 `LastPushTime` 为当前日期时间
    - Job推送时判断 `LastPushTime` 的日期部分，如果等于今天则跳过推送
    - 第二天自动恢复推送（因为日期不相等）
  - **风险等级推荐频率提示**：
    - 页面加载时获取用户风险等级
    - 根据风险等级显示推荐频率提示：
      - 高风险：建议每7天自查一次
      - 中风险：建议每14天左右自查一次
      - 低风险：每30天一次自查即可
    - 根据风险等级自动调整默认频率
  - **查看历史提醒**：
    - 提供"查看历史提醒"按钮，跳转到通知列表页

### 2.4 通知入口与角标联动

- **首页入口**：
  - 页面：`pages/Front/Index.vue`
  - 在功能卡片区域添加"消息通知"卡片
  - 显示未读通知角标（数字角标或红点）
  - 未读数量 > 99 时显示"99+"
  - 页面显示时（`onShow`）自动刷新未读数量
- **个人中心入口**：
  - 页面：`pages/Front/UserCenter.vue`
  - 在功能菜单中添加"消息通知"入口
  - 显示未读通知角标
  - 页面显示时自动刷新未读数量
- **未读数量接口**：
  - 接口路径：`POST /MessageNotice/UnreadCount`
  - 返回格式：`{ Code: "200", Success: true, Data: { Count: 5 } }`
  - 统计 `UserId` 下 `IsRead=false AND IsDelete=0` 的数量

## 三、后端实现说明（Spring Boot）

### 3.1 核心实体与 Mapper

- **消息通知表**：`MessageNotice`  
  - 关键字段：`Title, Content, Type, UserId, IsRead, IsDelete, PlanSendTime, ActualSendTime`  
  - 增强字段：`Priority`（优先级：0=普通，1=重要）、`IsPinned`（是否置顶：0=否，1=是）、`ExpireTime`（过期时间）、`ActionType`（前端点击行为类型）、`ActionParam`（行为参数）
- **个性化健康提醒表**：`TPersonalHealthRemind`  
  - 关键字段：`UserId, RemindType, RemindTime, RepeatFrequency, RemindStatus, LastPushTime`  
  - `LastPushTime` 用于"今天不再提醒"判断

### 3.2 前台业务 Service（`MessageNoticeService`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/MessageNoticeService.java`  
主要方法：

- `List(MessageNoticePagedInput)`  
  - 支持按 `UserId`、`Type`、`IsRead`、`StartTime`、`EndTime`、`KeyWord` 筛选
  - 默认排序：置顶优先 > 优先级 > 创建时间倒序
  - 返回分页结果 + 统计信息（Total/Read/Unread）
  - 自动过滤软删除记录和过期通知
- `Get(MessageNoticePagedInput)`  
  - 根据ID查询单条通知详情
- `CreateOrEdit(MessageNoticeDto)`  
  - 创建或更新通知记录（用于标记重要/置顶）
- `MarkRead(Integer id, Integer userId)`  
  - 标记单条通知为已读，校验用户归属
- `MarkAllRead(Integer userId)`  
  - 批量标记用户所有未读通知为已读
- `BatchSoftDelete(Integer userId, Boolean isRead)`  
  - 批量软删除通知（支持仅删除已读通知）
- `GetUnreadCount(Integer userId)`  
  - 获取用户未读通知数量（轻量级接口）

内部辅助方法：

- `BuilderQuery(MessageNoticePagedInput)`  
  - 构建查询条件，支持多维度筛选
  - 自动过滤软删除和过期通知
- `calculateStats(Integer userId, LocalDateTime startTime, LocalDateTime endTime)`  
  - 计算通知统计信息（总数/已读/未读）

### 3.3 个性化健康提醒 Service（`TPersonalHealthRemindService`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/TPersonalHealthRemindService.java`  
主要方法：

- `SnoozeToday(Integer userId, Integer remindType)`  
  - 更新指定提醒配置的 `LastPushTime` 为当前日期时间
  - Job推送时判断日期部分，如果等于今天则跳过推送

### 3.4 Job 推送策略（`PeriodicCheckupReminderJob` / `PersonalHealthRemindJob`）

位置：
- `HealthControl.springboot/src/main/java/com/example/web/jobs/PeriodicCheckupReminderJob.java`
- `HealthControl.springboot/src/main/java/com/example/web/jobs/PersonalHealthRemindJob.java`

**定期自查提醒Job**：
- 查询所有开启的定期自查提醒配置（`RemindType=0, RemindStatus=1`）
- 判断"今天不再提醒"（`LastPushTime` 的日期部分等于今天则跳过）
- 根据 `RepeatFrequency` 和 `LastScreenTime` 判断是否符合推送频率
- 获取用户风险等级，构建个性化推送内容：
  - 高风险：强调"结合就医建议，定期复查，建议每7天自查一次"
  - 中风险：强调"关注症状变化，建议每14天左右自查一次"
  - 低风险：强调"每30天一次自查即可"
- 创建通知记录，设置 `ActionType="GOTO_AUDIO_DETECT"`
- 更新 `LastPushTime` 为当前时间

**护嗓提醒Job**：
- 查询所有开启的护嗓提醒配置（`RemindType=1, RemindStatus=1`）
- 判断"今天不再提醒"（`LastPushTime` 的日期部分等于今天则跳过）
- 根据 `RemindTime` 判断是否到达提醒时间
- 创建通知记录，设置 `Type="VOICE_CARE"`
- 更新 `LastPushTime` 为当前时间

### 3.5 前台控制器（`MessageNoticeController` / `TPersonalHealthRemindController`）

位置：
- `HealthControl.springboot/src/main/java/com/example/web/controller/MessageNoticeController.java`
- `HealthControl.springboot/src/main/java/com/example/web/controller/TPersonalHealthRemindController.java`

主要职责：

- 暴露 `/MessageNotice/*` 前台 REST 接口：
  - `List`（通知列表，支持统计信息）
  - `Get`（通知详情）
  - `CreateOrEdit`（创建或更新通知）
  - `MarkRead`（标记已读）
  - `MarkAllRead`（批量标记已读）
  - `BatchSoftDelete`（批量软删除）
  - `UnreadCount`（未读数量）
- 暴露 `/TPersonalHealthRemind/*` 前台 REST 接口：
  - `SnoozeToday`（今天不再提醒）
- 统一封装响应为 `{ Code, Success, Msg, Data }`

## 四、前端实现说明（uni-app）

### 4.1 页面与路由

`HealthControl.uniapp/pages.json` 已注册如下消息通知相关页面：

- 通知列表页：`pages/Front/ReminderMessageList`  
- 通知详情页：`pages/Front/ReminderMessageDetail`  
- 提醒设置页：`pages/Front/HealthReminder`（已存在，已增强）

入口整合：

- `Index.vue`：首页增加"消息通知"功能卡片，显示未读角标，跳转 `/pages/Front/ReminderMessageList`；  
- `UserCenter.vue`：个人中心菜单中增加"消息通知"入口，显示未读角标，跳转 `/pages/Front/ReminderMessageList`；  
- `HealthReminder.vue`：提醒设置页提供"查看历史提醒"按钮，跳转到通知列表页。

### 4.2 HTTP 封装与登录态

- 统一使用 `utils/http.js` 中的 `Post` 方法调用后端接口，自动携带 `Authorization: Bearer <token>`；  
- `store` 中的 `commonStore` 提供 `UserId/UserInfo/Token` 等字段，页面在调用前根据是否登录决定：
  - 未登录：提示"请先登录"
  - 涉及用户数据的操作（查看通知、标记已读等）必须登录。

### 4.3 核心页面交互概要

- **ReminderMessageList**：
  - `onLoad` 生命周期中调用 `/MessageNotice/List` 拉取通知列表和统计信息
  - `onShow` 生命周期中刷新列表（从详情页返回时）
  - 支持搜索、日期筛选、类型筛选
  - 按日期分组展示（今天/近7天/更早）
  - 长按通知卡片弹出操作菜单
  - 点击通知自动标记已读并跳转
  - 滚动到底部自动加载下一页
- **ReminderMessageDetail**：
  - `onLoad` 时根据URL参数 `id` 加载通知详情
  - 自动标记为已读
  - 提供重要/置顶/删除操作按钮
  - 如有 `ActionType`，提供"跳转到相关页面"按钮
- **HealthReminder**：
  - `onLoad` 时加载用户提醒配置和风险等级
  - 根据风险等级显示推荐频率提示
  - 点击"今天不再提醒"调用 `/TPersonalHealthRemind/SnoozeToday`
- **Index / UserCenter**：
  - `onShow` 时调用 `/MessageNotice/UnreadCount` 获取未读数量
  - 显示角标（数字或红点）

### 4.4 跳转逻辑统一封装

通知点击跳转逻辑统一在 `ReminderMessageList.vue` 的 `handleClick` 方法中实现：

```javascript
const handleClick = async (item) => {
  // 标记已读
  if (!item.IsRead) {
    await Post('/MessageNotice/MarkRead', { Id: item.Id, UserId: UserId.value })
    item.IsRead = true
    // 更新统计
    if (stats.value) {
      stats.value.Read = (stats.value.Read || 0) + 1
      stats.value.Unread = Math.max(0, (stats.value.Unread || 0) - 1)
    }
  }

  // 根据 ActionType 跳转
  const actionType = (item.ActionType || '').toUpperCase()
  const actionParam = item.ActionParam

  // 如果ActionType为空，跳转到详情页
  if (!actionType || actionType === '') {
    uni.navigateTo({ url: `/pages/Front/ReminderMessageDetail?id=${item.Id}` })
    return
  }

  // 根据ActionType跳转到对应业务页面
  switch (actionType) {
    case 'GOTO_AUDIO_DETECT':
      uni.navigateTo({ url: '/pages/Front/AudioRecord' })
      break
    // ... 其他ActionType
  }
}
```

### 6.2 异常与兜底场景

- **无通知数据**：
  - 列表应展示"暂时还没有健康提醒"，而不是空白页
  - 统计卡片显示"0 条"
- **通知越权访问**：
  - 使用非本人的用户ID访问通知详情接口时，后端返回错误码
  - 前端展示统一错误提醒
- **搜索无结果**：
  - 搜索关键字无匹配时，列表显示空状态
  - 提示"未找到相关通知"
- **置顶数量超限**：
  - 尝试置顶第6条通知时，提示"最多只能置顶5条，请先取消部分置顶"
- **Job推送异常**：
  - Job执行时出现异常，应记录日志但不中断任务
  - 单个用户推送失败不影响其他用户

### 6.3 性能测试建议

1. **列表加载性能**：
   - 测试大量通知数据（1000+条）时的加载速度
   - 确认分页加载正常，不会一次性加载所有数据
   - 确认统计信息计算耗时在可接受范围内

2. **搜索性能**：
   - 测试关键字搜索的响应时间
   - 确认搜索不会导致数据库全表扫描

3. **Job推送性能**：
   - 测试大量用户（1000+）时的推送性能
   - 确认Job执行时间在合理范围内

### 6.4 边界测试

1. **时间边界**：

```java
// 判断"今天不再提醒"（LastPushTime的日期部分等于今天则跳过）
if (cfg.getLastPushTime() != null) {
```

```javascript
// 按日期分组
const todayItems = computed(() => {
  const today = new Date()
  today.setHours(0, 0, 0, 0)
  return items.value.filter(item => {
    const itemTime = new Date(item.CreationTime || item.PlanSendTime)
    itemTime.setHours(0, 0, 0, 0)
    return itemTime.getTime() === today.getTime()
  })
})

const weekItems = computed(() => {
  const today = new Date()
  today.setHours(0, 0, 0, 0)
  const weekAgo = new Date(today.getTime() - 7 * 24 * 60 * 60 * 1000)
  return items.value.filter(item => {
    const itemTime = new Date(item.CreationTime || item.PlanSendTime)
    itemTime.setHours(0, 0, 0, 0)
    return itemTime.getTime() < today.getTime() && itemTime.getTime() >= weekAgo.getTime()
  })
})
```
