# 消息通知中心模块 - 详细需求文档

## 一、模块概述

### 1.1 模块定位
消息通知中心模块是居家喉部健康自查小程序的核心服务闭环模块，旨在将"提醒配置页"升级为"**通知中心 + 行为闭环**"，实现通知的可追溯、可审计、可操作，形成完整的健康提醒服务链路。

### 1.2 核心价值
- **可追溯性**：记录什么时候提醒了什么、用户是否看过、点击后去了哪里
- **行为闭环**：通知点击后能正确跳转到对应业务页面，形成完整的用户行为链路
- **合规审计**：所有通知记录可追溯，可用于运营/医疗合规审计（不夸大、不恐吓、有据可查）
- **用户体验**：统一的通知管理入口，支持筛选、标记已读、清理等操作

### 1.3 与其他模块的关系
- **依赖模块**：
  - 用户基础模块（用户信息）
  - 健康档案模块（风险等级，用于推送策略）
  - 音频自查模块（检测记录，用于跳转）
  - 就医辅助模块（就医建议，用于跳转）
  - 饮食模块（饮食推荐，用于跳转）
- **支撑模块**：为所有业务模块提供统一的通知推送和用户触达能力
- **数据来源**：
  - `messagenotice`：通知记录表
  - `tpersonalhealthremind`：提醒配置表

## 二、功能需求详细说明

### 2.1 通知列表增强（ReminderMessageList.vue）

#### 2.1.1 功能描述
在现有通知列表页基础上，增强筛选、分组、操作和跳转功能，形成完整的通知中心。

#### 2.1.2 展示内容
1. **顶部统计卡片**
   - 最近30天收到提醒总数
   - 已阅读数量
   - 未阅读数量（计算得出）
   - 统计时间范围可配置（默认30天）

2. **筛选标签栏**
   - **全部**：显示所有通知
   - **未读**：仅显示未读通知（`IsRead = false`）
   - **自查提醒**：类型为 `CHECKUP` 的通知
   - **护嗓提醒**：类型为 `VOICE_CARE` 的通知
   - **随访提醒**：类型为 `FOLLOWUP` 的通知
   - **其他**：其他类型的通知（如 `ANNOUNCEMENT` 等）

3. **通知卡片列表**
   - **通知标题**：`Title` 字段
   - **通知内容**：`Content` 字段（最多显示2行，超出省略）
   - **通知类型标签**：不同颜色区分（自查=绿色，护嗓=蓝色，随访=橙色）
   - **未读标识**：红色圆点（`IsRead = false` 时显示）
   - **发送时间**：`PlanSendTime` 或 `ActualSendTime`（优先显示实际发送时间）
   - **操作按钮**：长按或右滑显示"删除"按钮（软删除）

4. **底部操作栏**
   - **全部标记为已读**：批量标记当前筛选结果中的所有未读通知为已读
   - **清理已读**：批量软删除所有已读通知（可选功能）

#### 2.1.3 交互功能
1. **点击通知卡片**
   - 自动标记为已读（调用 `/MessageNotice/MarkRead`）
   - 根据 `ActionType` 和 `ActionParam` 跳转到对应页面：
     - `GOTO_AUDIO_DETECT` → 跳转到音频自查页 `/pages/Front/AudioRecord`
     - `GOTO_HEALTH_RECORD` → 跳转到健康档案页 `/pages/Front/HealthRecord`
     - `GOTO_SYMPTOM_FORM` → 跳转到症状记录表单页 `/pages/Front/SymptomLogForm`
     - `GOTO_DIET_HOME` → 跳转到饮食首页 `/pages/Front/DietHome`
     - `GOTO_MEDICAL_HOME` → 跳转到就医辅助首页 `/pages/Front/MedicalHome`
     - `GOTO_MEDICAL_RECOMMEND_DETAIL` → 跳转到就医建议详情页，参数为 `RecommendId`（从 `ActionParam` 解析）
     - `GOTO_AUDIO_RESULT` → 跳转到检测结果详情页，参数为 `DetectId`（从 `ActionParam` 解析）
     - 无 `ActionType` 或未知类型 → 弹出模态框显示通知内容

2. **筛选切换**
   - 点击筛选标签，立即刷新列表
   - 切换筛选条件时，重置分页（`Page = 1`）

3. **分页加载**
   - 滚动到底部自动加载下一页
   - 每页20条记录
   - 加载中显示"加载中..."提示

4. **批量操作**
   - **全部标记为已读**：弹出确认弹窗，确认后调用 `/MessageNotice/MarkAllRead`
   - **清理已读**（可选）：弹出确认弹窗，确认后批量软删除已读通知

#### 2.1.4 业务规则
1. **未读标识**：`IsRead = false` 时显示红色圆点
2. **时间显示**：优先显示 `ActualSendTime`，如为空则显示 `PlanSendTime`
3. **时间格式**：`MM-DD HH:mm`（如：03-07 09:00）
4. **内容截断**：通知内容超过2行时，显示省略号
5. **空状态**：无通知时显示"暂时还没有健康提醒"

### 2.2 提醒设置增强（HealthReminder.vue）

#### 2.2.1 功能描述
在现有提醒设置页基础上，增强"今天不再提醒"功能和风险等级推荐频率提示。

#### 2.2.2 展示内容
1. **提醒总览卡片**
   - 总开关状态（已开启/已关闭）
   - 各类型提醒的当前配置摘要：
     - 自查提醒：频率 + 时间（如：每周一次 · 09:00）
     - 护嗓提醒：时间（如：21:00 护嗓贴士）
     - 随访提醒：状态（如：风险变化时提醒复查）

2. **定期自查提醒卡片**
   - 开关：`RemindStatus`（0=关闭，1=开启）
   - 提醒频率：`RepeatFrequency`（0=每天，1=每周一至五，2=每周一次，3=每30天一次）
   - 提醒时间：`RemindTime`（时间选择器，格式：HH:mm）
   - **风险等级推荐提示**（新增）：
     - 高风险：显示"当前为高风险，建议每 7 天自查一次，并结合就医建议。"
     - 中风险：显示"当前为中风险，建议每 14 天左右自查一次。"
     - 低风险：显示"当前为低风险，每 30 天一次自查即可。"
     - 未评定：不显示推荐提示

3. **护嗓饮食/用声提醒卡片**
   - 开关：`RemindStatus`
   - 提醒时间：`RemindTime`
   - **"今天不再提醒"功能**（新增）：
     - 点击后，调用后端接口记录"今天不再提醒"状态
     - 后端根据 `LastPushTime` 和当前日期判断，今天不再推送该类型提醒

4. **就医/复查提醒卡片**
   - 开关：`RemindStatus`
   - 提示文案：系统会根据您的风险等级与最近一次检测时间，适时推送随访提醒。

#### 2.2.3 交互功能
1. **保存设置**
   - 点击"保存设置"按钮，调用 `/TPersonalHealthRemind/CreateOrEdit` 保存所有提醒配置
   - 保存成功后显示"已保存"提示

2. **"今天不再提醒"**
   - 点击后，调用后端接口 `/TPersonalHealthRemind/SnoozeToday`
   - 后端更新 `LastPushTime` 为当前日期时间，Job 推送时判断今天不再推送
   - 前端显示"今天剩余时间不再提醒"提示

3. **风险等级推荐频率**
   - 页面加载时，调用 `/TPersonalLaryngealHealthRecord/Get` 获取用户风险等级
   - 根据风险等级自动调整默认频率：
     - 高风险 → 默认选择"每天一次"
     - 中风险 → 默认选择"每周一次"
     - 低风险 → 默认选择"每 30 天一次"

4. **查看历史提醒**
   - 点击"查看历史提醒"按钮，跳转到通知列表页 `/pages/Front/ReminderMessageList`

#### 2.2.4 业务规则
1. **"今天不再提醒"逻辑**：
   - 后端记录 `LastPushTime` 为当前日期时间（如：2026-03-07 10:00:00）
   - Job 推送时，判断 `LastPushTime` 的日期部分是否等于今天
   - 如果等于今天，则跳过本次推送
   - 第二天自动恢复推送（因为日期不相等）

2. **风险等级推荐频率**：
   - 仅在用户首次进入页面或风险等级变化时显示推荐提示
   - 用户手动修改频率后，不再自动调整

### 2.4 Job 推送策略增强

#### 2.4.1 推送频率控制
**现有逻辑**：
- 根据 `RemindTime` 和 `RepeatFrequency` 判断是否需要推送

**需要增强**：
- 结合 `LastPushTime` 和用户风险等级，控制推送频率
- **"今天不再提醒"判断**：
  - 如果 `LastPushTime` 的日期部分等于今天，则跳过本次推送
- **风险等级频率控制**：
  - 高风险用户：推送频率可适当提高（如每天一次自查提醒）
  - 中风险用户：正常频率（如每周一次自查提醒）
  - 低风险用户：降低频率（如每30天一次自查提醒）

#### 2.4.2 推送内容个性化
**根据风险等级调整推送内容**：
- **高风险用户**：
  - 自查提醒：强调"结合就医建议，定期复查"
  - 随访提醒：增加"建议尽快就医"的提示
- **中风险用户**：
  - 自查提醒：强调"关注症状变化"
  - 随访提醒：提示"必要时前往门诊复查"
- **低风险用户**：
  - 自查提醒：强调"保持良好习惯"
  - 随访提醒：不推送或降低频率

### 2.5 通知详情与置顶/重要标记

#### 2.5.1 功能描述
在现有通知卡片基础上，支持\"重要\"与\"置顶\"标记，以及通知详情页，方便用户保留关键提醒。

#### 2.5.2 展示与交互
1. **通知卡片右上角标识**  
   - 重要通知：在标题左侧显示\"重要\"小标签（橙色）  
   - 置顶通知：卡片右上角显示📌图标，永远排在列表顶部（按时间再排序）

2. **操作方式**  
   - 长按通知卡片 → 弹出操作菜单：`标记为重要 / 取消重要 / 置顶 / 取消置顶 / 删除`  
   - 选择\"置顶\"后：前端将该通知移动到列表顶部，后端持久化 `IsPinned = 1`  
   - 选择\"标记为重要\"后：前端展示\"重要\"标签，后端更新 `Priority = 1`

3. **通知详情页（可选）**  
   - 新增 `/pages/Front/ReminderMessageDetail.vue`：  
     - 展示完整标题、时间、类型、正文全文  
     - 显示重要/置顶状态  
     - 下方提供操作按钮：`标记为重要 / 置顶 / 删除 / 跳转到相关页面`  
   - 通知卡片点击可直接跳详情，或保留当前\"直接跳业务页\"逻辑，由 `ActionType` 决定：  
     - 若 `ActionType` 为空 → 进入通知详情页  
     - 若有 `ActionType` → 直接跳业务页（当前行为不变）

#### 2.5.3 业务规则
1. 重要通知：  
   - 使用 `Priority` 字段：`0=普通，1=重要`  
   - 列表默认按 `IsPinned DESC, Priority DESC, CreationTime DESC` 排序。
2. 置顶通知：  
   - 使用 `IsPinned` 字段：`0=未置顶，1=置顶`  
   - 置顶数量建议前端限制在 5 条以内（超过时提示用户先取消部分置顶）。

### 2.6 通知搜索与分组展示

#### 2.6.1 功能描述
为方便用户在通知较多时快速定位信息，增加按关键字与日期范围搜索，并对列表按日期分组展示。

#### 2.6.2 前端交互
1. **搜索框**  
   - 位于筛选标签栏上方或右侧：输入标题/内容关键字  
   - 输入关键字后点击键盘\"搜索\"或右侧图标，重新调用 `/MessageNotice/List`，增加 `Keyword` 参数。

2. **日期筛选**  
   - 在筛选条右侧增加\"时间\"按钮，点击弹出：`全部 / 近7天 / 近30天 / 自定义`  
   - 前端根据选择设置 `StartTime` / `EndTime` 再发起列表请求。

3. **按日期分组展示**  
   - 列表中以\"今天\"、\"近7天\"、\"更早\"分割区域标题显示：  
     - 今天（CreationTime 日期=今天）  
     - 近7天（不含今天，最近7日内）  
     - 更早（其余时间）  
   - 仅前端分组渲染，后端仍按时间排序返回。

#### 2.6.3 后端支持
1. 在 `MessageNoticePagedInput` 中新增可选字段：`Keyword`（用于按标题/内容模糊搜索）。  
2. `BuilderQuery` 中增加：当 `Keyword` 不为空时，对 `Title`、`Content` 做 `LIKE` 查询。  
3. 已在 2.3.1 中覆盖时间范围筛选参数 `StartTime/EndTime`。

### 2.7 通知入口与角标联动

#### 2.7.1 功能描述
在首页、个人中心或底部导航上显示未读通知角标，提升通知触达效率。

#### 2.7.2 前端行为
1. 小程序启动或从后台返回前台时：  
   - 轻量调用 `/MessageNotice/List`，仅取 `Limit=1` 且 `IsRead=false`，或新增 `/MessageNotice/UnreadCount` 接口。  
   - 若未读数量 > 0，在\"消息/提醒\"入口显示红点或数字角标（最多显示为 `99+`）。

2. 进入通知列表页并成功加载后：  
   - 若当前筛选为\"全部\"或\"未读\"，可根据接口返回的未读数动态更新角标（清零或减少）。

#### 2.7.3 可选接口（如果不想重用 List）
**接口路径**：`POST /MessageNotice/UnreadCount`  
**请求参数**：`{ "UserId": 2 }`  
**响应数据**：`{ "Code": "200", "Success": true, "Data": { "Count": 5 } }`  
实现可简单统计 `UserId` 下 `IsRead=false AND IsDelete=0` 的数量。

## 三、数据表设计

### 3.1 消息通知表（messagenotice）

**表结构**（基于现有表，无需修改）：

| 字段名 | 类型 | 说明 | 必填 | 备注 |
|--------|------|------|------|------|
| Id | Integer | 主键ID | 是 | 自增 |
| CreationTime | LocalDateTime | 创建时间 | 是 | 自动填充 |
| Title | String(64) | 通知标题 | 是 | |
| Content | String(512) | 通知内容 | 是 | |
| Type | String(32) | 通知类型 | 是 | CHECKUP/VOICE_CARE/FOLLOWUP/ANNOUNCEMENT等 |
| TargetKey | String(64) | 目标业务主键 | 否 | 如检测Id/档案Id |
| PlanSendTime | LocalDateTime | 计划发送时间 | 否 | |
| ActualSendTime | LocalDateTime | 实际发送时间 | 否 | |
| UserId | Integer | 接收用户Id | 是 | 外键，关联AppUser表 |
| IsSend | Boolean | 是否已触发发送 | 是 | 0=未发送，1=已发送 |
| IsSuccess | Boolean | 发送是否成功 | 是 | 0=失败，1=成功 |
| IsRead | Boolean | 是否已读 | 是 | 0=未读，1=已读 |
| ResultMsg | String(255) | 发送结果说明 | 否 | |
| ActionType | String(32) | 前端点击行为类型 | 否 | GOTO_AUDIO_DETECT等 |
| ActionParam | String(128) | 行为参数 | 否 | 如DetectId/RecommendId 等 |
| RelationNo | String(64) | 关联单号 | 否 | 预留 |
| Priority | TinyInt(1) | 优先级 | 是 | 0=普通，1=重要，预留更高等级 |
| IsPinned | TinyInt(1) | 是否置顶 | 是 | 0=否，1=是 |

public PagedResult<MessageNoticeDto> List(MessageNoticePagedInput input) {
    LambdaQueryWrapper<MessageNotice> queryWrapper = BuilderQuery(input);
    
    // 时间范围筛选
    if (input.getStartTime() != null) {
        queryWrapper.ge(MessageNotice::getPlanSendTime, input.getStartTime());
    }
    if (input.getEndTime() != null) {
        queryWrapper.le(MessageNotice::getPlanSendTime, input.getEndTime());
    }
    
    // 分页查询
    Page<MessageNotice> page = new Page<>(input.getPage(), input.getLimit());
    IPage<MessageNotice> pageResult = messageNoticeMapper.selectPage(page, queryWrapper);
    
    // 转换为DTO
    List<MessageNoticeDto> items = pageResult.getRecords().stream()
        .map(entity -> {
            try {
                return entity.MapToDto();
            } catch (Exception e) {
                return null;
            }
        })
        .filter(Objects::nonNull)
        .collect(Collectors.toList());
    
    // 计算统计信息
    MessageNoticeStats stats = calculateStats(input.getUserId(), input.getStartTime(), input.getEndTime());
    
    PagedResult<MessageNoticeDto> result = new PagedResult<>();
    result.setItems(items);
    result.setTotalCount((int) pageResult.getTotal());
    result.setStats(stats);  // 新增统计信息
    
    return result;
}

// 新增统计方法
private MessageNoticeStats calculateStats(Integer userId, LocalDateTime startTime, LocalDateTime endTime) {
    LambdaQueryWrapper<MessageNotice> queryWrapper = Wrappers
        .<MessageNotice>lambdaQuery()
        .eq(MessageNotice::getUserId, userId)
        .eq(MessageNotice::getIsDelete, false);
    
    if (startTime != null) {
        queryWrapper.ge(MessageNotice::getPlanSendTime, startTime);
    }
    if (endTime != null) {
        queryWrapper.le(MessageNotice::getPlanSendTime, endTime);
    }
    
    List<MessageNotice> all = messageNoticeMapper.selectList(queryWrapper);
    int total = all.size();
    int read = (int) all.stream().filter(MessageNotice::getIsRead).count();
    int unread = total - read;
    
    MessageNoticeStats stats = new MessageNoticeStats();
    stats.setTotal(total);
    stats.setRead(read);
    stats.setUnread(unread);
    return stats;
}

## 六、业务流程

### 6.1 通知列表查看流程

```
用户进入通知列表页
    ↓
调用 /MessageNotice/List（带UserId和筛选条件）
    ↓
后端返回通知列表 + 统计信息
    ↓
前端展示列表和统计
    ↓
用户点击通知卡片
    ↓
自动调用 /MessageNotice/MarkRead 标记已读
    ↓
根据 ActionType 和 ActionParam 跳转到对应页面
    ├─ GOTO_AUDIO_DETECT → 音频自查页
    ├─ GOTO_HEALTH_RECORD → 健康档案页
    ├─ GOTO_MEDICAL_HOME → 就医辅助首页
    └─ 其他 → 弹出内容模态框
```

### 6.2 "今天不再提醒"流程

```
用户在提醒设置页点击"今天不再提醒"
    ↓
调用 /TPersonalHealthRemind/SnoozeToday
    ↓
后端更新 LastPushTime = 当前日期时间
    ↓
Job 推送时判断 LastPushTime 的日期部分
    ├─ 等于今天 → 跳过推送
    └─ 不等于今天 → 正常推送
    ↓
第二天自动恢复推送（因为日期不相等）
```

### 6.3 通知推送流程（增强）

```
Job 定时任务触发（如每天09:00）
    ↓
查询所有开启的提醒配置
    ↓
遍历每个提醒配置
    ↓
判断"今天不再提醒"（LastPushTime日期 == 今天？）
    ├─ 是 → 跳过
    └─ 否 → 继续
    ↓
判断推送频率（RepeatFrequency）
    ├─ 不符合 → 跳过
    └─ 符合 → 继续
    ↓
获取用户风险等级
    ↓
构建个性化推送内容
    ↓
创建通知记录（MessageNotice）
    ↓
更新 LastPushTime = 当前时间
    ↓
标记通知为已发送（IsSend = true）
```

// 判断今天是否已推送过
private boolean isSnoozedToday(TPersonalHealthRemind remind) {
    if (remind.getLastPushTime() == null) {
        return false;
    }
    LocalDate lastPushDate = remind.getLastPushTime().toLocalDate();
    LocalDate today = LocalDate.now();
    return lastPushDate.equals(today);
}

#### 8.2.2 推送频率判断
```java
// 判断是否符合推送频率
private boolean shouldPush(TPersonalHealthRemind remind) {
    Integer frequency = remind.getRepeatFrequency();
    LocalDateTime now = LocalDateTime.now();
    LocalTime remindTime = LocalTime.parse(remind.getRemindTime());
    
    switch (frequency) {
        case 0:  // 每天一次
            return now.toLocalTime().isAfter(remindTime);
        case 1:  // 每周一至五
            DayOfWeek dayOfWeek = now.getDayOfWeek();
            return (dayOfWeek.getValue() >= 1 && dayOfWeek.getValue() <= 5) 
                && now.toLocalTime().isAfter(remindTime);
        case 2:  // 每周一次
            // 判断是否距离上次推送超过7天
            if (remind.getLastPushTime() != null) {
                long days = ChronoUnit.DAYS.between(remind.getLastPushTime(), now);
                return days >= 7 && now.toLocalTime().isAfter(remindTime);
            }
            return now.toLocalTime().isAfter(remindTime);
        case 3:  // 每30天一次
            if (remind.getLastPushTime() != null) {
                long days = ChronoUnit.DAYS.between(remind.getLastPushTime(), now);
                return days >= 30 && now.toLocalTime().isAfter(remindTime);
            }
            return now.toLocalTime().isAfter(remindTime);
        default:
            return false;
    }
}
```
