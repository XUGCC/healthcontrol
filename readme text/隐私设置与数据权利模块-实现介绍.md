## 隐私设置与数据权利模块 - 实现介绍文档

## 一、模块概述

隐私设置与数据权利模块围绕“**隐私偏好配置 + 数据导出/删除申请 + 模型优化授权透明度 + 数据留存与自动清理**”四条主线，为居家喉部健康自查小程序提供合规的数据闭环能力。  

本模块已经完成以下几块能力：

- **隐私设置主页面**：支持本地存储开关、数据匿名授权总开关、隐私协议查看与同意、数据留存与自动清理策略配置，以及数据权利入口。
- **数据导出/删除申请**：用户可自主发起导出/删除数据的申请，系统以任务形式记录并异步处理。
- **数据权利申请记录**：用户可查看导出/删除申请的历史记录与处理状态，并在导出完成后直接下载文件。
- **模型优化授权明细**：用户可集中查看“已授权用于模型优化的标注记录”，并对单条记录收回授权。
- **数据留存与自动清理**：用户可以设置历史数据保留时长，后端定时任务按策略对过期数据做软删除。
- **管理端配置与审计**：管理员可在后台查看和导出用户隐私设置、数据导出申请、数据删除申请等信息，并通过统计卡片了解整体隐私配置情况。

后端基于 **Spring Boot + MyBatis-Plus**，前端基于 **uni-app（微信小程序/H5）**，管理端为 **Element Plus**。接口统一通过 `GlobalResponseAdvice` 封装为：

```json
{
  "Code": "200",
  "Success": true,
  "Msg": "成功",
  "Data": { ... }
}
```

## 二、主要实现功能概览

### 2.1 隐私设置主页面（PrivacySetting）

- **前台入口**
  - 在 `UserCenter.vue` 中新增“隐私设置”菜单项，点击跳转到 `pages/Front/PrivacySetting`。
- **页面结构**
  - 本地存储开关：控制是否在设备本地缓存音频、报告等数据，关闭时弹出二次确认。
  - 数据留存与自动清理：
    - 开关：是否开启自动清理（`DataRetentionEnabled`）。
    - 保留时长选择：3/6/12 个月或永久（`DataRetentionMonths`）。
    - 开/关时均有风险提示弹窗。
  - 数据匿名授权总开关：控制用户是否同意将脱敏标注数据用于模型优化，开关变更有确认说明。
  - 隐私协议：展示当前是否已同意，提供“查看隐私协议”和“一键同意”操作。
  - 数据权利入口：跳转到数据导出申请页、数据删除申请页、数据权利申请记录页。
- **核心交互**
  - 页面展示时调用 `/Front/Privacy/GetSetting` 读取当前用户隐私设置，没有记录时后端自动创建默认记录。
  - 每次开关或设置变更时调用 `/Front/Privacy/UpdateSetting` 实时保存，并以 Toast 提示结果。
  - 同意隐私协议时调用 `/Front/Privacy/AgreeProtocol`，后端写入 `PrivacyAgreeStatus=true`。

### 2.2 数据导出申请（DataExportRequest）

- **页面入口**
  - 从 `PrivacySetting` 中“申请导出数据”跳转到 `pages/Front/DataExportRequest`。
- **主要能力**
  - 导出数据类型多选：音频自查记录、健康档案记录、症状日志、饮食记录、确诊标注记录、问卷答题记录、健康提醒设置、消息通知记录。
  - 导出格式选择：当前 MVP 实现仅支持 `JSON`，前端文案中已说明后续可扩展 Excel/PDF。
  - 申请说明（可选）：记录用户期望或补充说明。
  - 提交后创建导出任务记录，并提示是否立即下载导出文件。
- **接口**
  - `POST /Front/Privacy/ExportRequest`：创建导出申请记录，返回 `RequestId`。
  - `GET /Front/Privacy/DownloadExport?RequestId=...`：根据申请记录实时生成 JSON 文件并返回文件流。

### 2.3 数据删除申请（DataDeleteRequest）

- **页面入口**
  - 从 `PrivacySetting` 中“申请删除数据”跳转到 `pages/Front/DataDeleteRequest`。
- **主要能力**
  - 顶部醒目警告：提示删除操作不可恢复，且已用于模型训练的数据难以从历史模型中完全移除。
  - 删除数据类型多选：与导出类似，增加“账户信息（暂不支持自动删除）”说明。
  - 删除原因选择 + 其他原因补充说明。
  - 必须勾选“我已了解删除操作的后果，确认提交删除申请”复选框。
- **接口**
  - `POST /Front/Privacy/DeleteRequest`：创建删除申请记录，初始状态为 `Pending`。

### 2.4 数据权利申请记录（DataRightsHistory）

- **页面入口**
  - 从 `PrivacySetting` 中“申请记录”跳转到 `pages/Front/DataRightsHistory`。
- **主要能力**
  - 顶部标签切换：全部/导出申请/删除申请。
  - 列表聚合展示导出与删除申请记录：申请时间、数据类型描述、当前状态。
  - 对于导出申请且状态为 `Completed` 的记录，提供“下载”按钮，直连后端下载接口。
  - 实现下拉刷新与上拉加载更多，支持较长历史的翻阅。
- **接口**
  - `POST /Front/Privacy/ExportRequestList`
  - `POST /Front/Privacy/DeleteRequestList`
  - `GET /Front/Privacy/DownloadExport`

### 2.5 隐私协议查看页（PrivacyProtocol）

- **页面入口**
  - 从 `PrivacySetting` 点击“查看隐私协议”跳转到 `pages/Front/PrivacyProtocol`。
- **主要能力**
  - 展示隐私协议摘要内容（正式版可替换为配置化富文本）。
  - 提供“我已阅读并同意”按钮，同样调用 `/Front/Privacy/AgreeProtocol`。

### 2.6 模型优化授权明细（AuthUsageDetail）

- **页面入口**
  - 从 `PrivacySetting` 中“查看授权明细”跳转到 `pages/Front/AuthUsageDetail`。
- **主要能力**
  - 顶部授权状态筛选：全部 / 已授权 / 未授权。
  - 时间范围筛选：全部 / 近 7 天 / 近 30 天 / 自定义日期。
  - 检测结果筛选：全部 / 良性 / 恶性 / 其他（前端基于结果文案匹配）。
  - 列表项：展示检测时间、初筛结果、医院确诊结果、授权状态、标注说明。
  - 已授权记录提供“取消授权”按钮，仅支持单条收回，不提供“一键全部收回”。
- **接口**
  - `POST /Front/ModelLabel/List`：获取当前用户的标注列表。
  - `POST /Front/ModelLabel/UpdateAuth`：单条更新授权状态。

### 2.7 数据留存与自动清理（DataRetentionSetting）

- **前台表现**
  - 在 `PrivacySetting` 页面展示自动清理开关和“保留最近 X 个月”的描述文案。
  - 用户首次开启自动清理时弹出确认弹窗，说明过期数据将被软删除且可能影响趋势分析。
- **后台行为**
  - `PrivacyDataRetentionJob` 每天凌晨按用户配置进行扫描，对早于阈值时间的数据记录做软删除。

## 三、后端实现说明（Spring Boot）

### 3.1 相关表与 Mapper

- 用户隐私设置表：`tuserprivacysetting` + `TUserPrivacySettingMapper`
  - 字段：`LocalStorageStatus`、`DataAnonymousAuth`、`PrivacyAgreeStatus`、`DataRetentionEnabled`、`DataRetentionMonths`、`IsDelete` 等。
  - 每个用户业务上只应有一条记录，`Service` 层在创建/更新时会按 `UserId` 去重。
- 数据导出申请表：`tdataexportrequest` + `TDataExportRequestMapper`
  - 字段：`DataType`（JSON 数组字符串）、`ExportFormat`、`Status`、`FileUrl`、`FileSize`、`ExpireTime`、`ProcessTime`、`ErrorMessage` 等。
- 数据删除申请表：`tdatadeleterequest` + `TDataDeleteRequestMapper`
  - 字段：`DataType`、`DeleteReason`、`Status`、`DeleteCount`、`ProcessTime`、`ErrorMessage` 等。
- 其他业务数据表（只列出与隐私模块有直接互动的）：
  - `TAudioScreenRecord`：音频自查记录。
  - `TPersonalLaryngealHealthRecord`：个人健康档案。
  - `TSymptomLog`：症状日志。
  - `TUserDietRecord`：饮食记录。
  - `TModelOptimizeLabel`：模型优化标注记录。
  - `TPersonalHealthRemind`：个性化健康提醒。
  - `MessageNotice`：系统/提醒消息。
  - `TUserQuestionnaireAnswer`：问卷答题。

### 3.2 前台隐私接口 Controller（`FrontPrivacyController`）

位置：`HealthControl.springboot/src/main/java/com/example/web/controller/FrontPrivacyController.java`  

主要接口：

- `POST /Front/Privacy/GetSetting`
  - 从 `BaseContext` 中获取当前登录用户 ID。
  - 通过 `TUserPrivacySettingService.Get` 按 `UserId` 查询隐私设置。
  - 若没有记录，则创建一条默认记录（全部关闭，未同意协议，未开启自动清理）。
- `POST /Front/Privacy/UpdateSetting`
  - 校验当前用户身份，只允许更新自己的设置。
  - 调用 `TUserPrivacySettingService.CreateOrEdit` 完成插入/更新。
- `POST /Front/Privacy/AgreeProtocol`
  - 获取或创建隐私设置记录，将 `PrivacyAgreeStatus` 置为 `true` 并保存。
  - 返回简单的 `{ Success, Msg }` 结果。
- `POST /Front/Privacy/ExportRequest`
  - 入参为 `FrontPrivacyExportRequestInput`，由 `FrontPrivacyService.exportRequest` 处理。
- `POST /Front/Privacy/ExportRequestList`
  - 入参为 `FrontPrivacyExportRequestListInput`，返回 `PagedResult<TDataExportRequestDto>`。
- `GET /Front/Privacy/DownloadExport`
  - 校验 `RequestId` 所属用户与当前登录用户一致。
  - 调用 `FrontPrivacyService.downloadExport` 实时组装导出 JSON。
- `POST /Front/Privacy/DeleteRequest`
  - 入参 `FrontPrivacyDeleteRequestInput`，由 `FrontPrivacyService.deleteRequest` 新增一条 `Pending` 删除任务。
- `POST /Front/Privacy/DeleteRequestList`
  - 入参 `FrontPrivacyDeleteRequestListInput`，用于用户端查看个人删除申请记录。

### 3.3 隐私设置 Service（`TUserPrivacySettingServiceImpl`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/impl/TUserPrivacySettingServiceImpl.java`  

关键点：

- `BuilderQuery`：支持按 `UserId`、`LocalStorageStatus`、`DataAnonymousAuth`、`PrivacyAgreeStatus`、`DataRetentionEnabled`、`DataRetentionMonths` 等字段组合查询。
- `List`：管理端分页列表，默认按 `CreationTime` 倒序，并填充关联的用户信息 `UserDto`。
- `Get`：前台/后台单条查询，支持按 `Id` 或 `UserId` 取第一条匹配记录。
- `CreateOrEdit`：
  - 如果 `Id` 为空，则尝试按 `UserId` 查询现有记录，有则更新，无则插入。
  - 始终确保一个用户只有一条有效的隐私设置记录。

### 3.4 数据导出/删除前台 Service（`FrontPrivacyService`）

位置：`HealthControl.springboot/src/main/java/com/example/web/service/front/FrontPrivacyService.java`  

- `exportRequest(FrontPrivacyExportRequestInput input)`：
  - 校验当前登录用户。
  - 校验至少选择一种导出数据类型。
  - 创建 `TDataExportRequest` 记录，MVP 实现中 `Status` 直接设为 `Completed`，表示用户可立即下载。
- `exportRequestList(FrontPrivacyExportRequestListInput input)`：
  - 按当前用户 + 非删除记录 + 状态筛选，返回分页结果。
- `downloadExport(Integer requestId, HttpServletResponse response)`：
  - 校验 `RequestId` 归属与权限。
  - 根据 `DataType` 解析需要导出的数据范围，对多张业务表进行查询并脱敏（如姓名/手机号/邮箱）。
  - 组装为 JSON 对象写入响应输出流，前端通过 `uni.downloadFile` + `uni.saveFile` 保存。
- `deleteRequest(FrontPrivacyDeleteRequestInput input)`：
  - 校验当前用户，校验至少选择一种删除数据类型。
  - 新增 `TDataDeleteRequest` 记录，初始 `Status=Pending`，等待后台定时任务异步处理。
- `deleteRequestList(FrontPrivacyDeleteRequestListInput input)`：
  - 按当前用户 + 非删除记录 + 状态筛选，返回分页结果。

### 3.5 数据删除定时任务（`PrivacyDeleteRequestJob`）

位置：`HealthControl.springboot/src/main/java/com/example/web/tools/jobs/PrivacyDeleteRequestJob.java`  

- 每 1 分钟执行一次 `processPendingDeleteRequests`：
  - 选出最多 3 条 `Status=Pending` 且未软删除的 `TDataDeleteRequest`。
  - 将任务状态更新为 `Processing`。
  - 根据 `DataType` 解析需要处理的业务表，对对应用户的数据执行软删除（`IsDelete=true`）。
  - 统计实际删除记录数，写回 `DeleteCount`、`ProcessTime`、`Status=Completed`。
  - 出错时写入 `ErrorMessage` 并将 `Status` 置为 `Failed`。

### 3.6 数据留存定时任务（`PrivacyDataRetentionJob`）

位置：`HealthControl.springboot/src/main/java/com/example/web/tools/jobs/PrivacyDataRetentionJob.java`  

- 每天凌晨 3:30 执行 `cleanupByRetentionSetting`：
  - 查询所有 `DataRetentionEnabled=true` 且 `DataRetentionMonths>0` 的用户隐私设置。
  - 针对每个用户计算保留阈值时间 `now - months`。
  - 分别对音频记录、健康档案、症状日志、饮食记录、健康提醒、消息通知、问卷答题等表按阈值进行软删除。
  - 模型优化标注记录出于合规与建模考虑，默认不参与自动清理，可在后续版本中按业务需要调整。

### 3.7 模型标注与隐私联动（`TModelOptimizeLabelServiceImpl`）

- 在前台创建/更新标注时（`CreateOrEditForFront`）：
  - 若用户对本条标注设置 `AuthStatus=true`，服务层会调用内部 `checkAndUpdatePrivacySetting`，自动将该用户的 `DataAnonymousAuth` 开启，避免出现“单条授权但总开关关闭”的矛盾状态。
- 在前台修改授权状态（`UpdateAuthForFront`）：
  - 只允许当前用户操作自己的标注。
  - 当授权从未授权改为已授权时，同样会联动开启匿名授权总开关。

## 四、前端实现说明（uni-app & ElementUI）

### 4.1 小程序页面与路由

`HealthControl.uniapp/pages.json` 已注册隐私相关页面：

- `pages/Front/PrivacySetting`
- `pages/Front/DataExportRequest`
- `pages/Front/DataDeleteRequest`
- `pages/Front/PrivacyProtocol`
- `pages/Front/DataRightsHistory`
- `pages/Front/AuthUsageDetail`

个人中心入口：

- `UserCenter.vue`：新增“隐私设置”菜单项，点击跳转 `/pages/Front/PrivacySetting`。

### 4.2 小程序端 HTTP 与登录态复用

- HTTP 封装：`HealthControl.uniapp/utils/http.js`
  - 使用统一 `Post` 方法，自动附带 `Authorization` 头。
- 登录态管理：`store/index.js` 的 `commonStore`
  - 提供 `Token/UserInfo/RoleType/UserId` 等计算属性。
  - 前端隐私与数据权利接口均在登录后才能使用。

### 4.3 具体页面实现

- `PrivacySetting.vue`
  - 在 `onShow` 时调用 `/Front/Privacy/GetSetting` 初始化 `formData`。
  - 针对每个开关（本地存储、匿名授权、自动清理）封装 `saveSetting` 方法，调用 `/Front/Privacy/UpdateSetting` 后更新本地状态并 Toast 提示。
  - 同意协议按钮调用 `/Front/Privacy/AgreeProtocol`，成功后刷新设置。
  - “申请导出/删除/申请记录/授权明细”分别跳转到对应页面。

- `DataExportRequest.vue`
  - 使用 `checkbox-group` 让用户多选导出数据类型。
  - 当前仅提供 `JSON` 选项作为导出格式。
  - 提交时调用 `/Front/Privacy/ExportRequest`，成功后弹窗询问“是否立即下载”，如确认则拼接 `DownloadExport` 链接并通过 `uni.downloadFile` 下载。

- `DataDeleteRequest.vue`
  - 顶部以醒目颜色块展示删除风险提示。
  - 用户需勾选至少一个删除数据类型并勾选确认复选框。
  - 提交时调用 `/Front/Privacy/DeleteRequest`，成功后跳转到申请记录页。

- `DataRightsHistory.vue`
  - 使用顶部 tab 切换“全部/导出申请/删除申请”并通过 `computed` 聚合两个列表。
  - 使用 `onShow` 触发首次加载，`onPullDownRefresh` 实现下拉刷新，`onReachBottom` 实现上拉分页加载。
  - 对导出类型记录提供“下载”按钮，内部逻辑与导出页下载一致。

- `PrivacyProtocol.vue`
  - 展示简要版隐私协议内容和说明。
  - “我已阅读并同意”按钮直接调用 `/Front/Privacy/AgreeProtocol`，成功后返回上一页。

- `AuthUsageDetail.vue`
  - 使用本地状态组合 `authFilter/timeFilter/resultFilter` 实现前端多维筛选。
  - 列表数据来源于 `/Front/ModelLabel/List`，仅返回当前用户的标注记录，并自动过滤软删除。
  - “取消授权”按钮调用 `/Front/ModelLabel/UpdateAuth` 并刷新列表。

### 4.4 管理端页面（Element Plus）

- `TUserPrivacySettingList.vue`
  - 提供按用户、本地存储状态、匿名授权状态、协议同意状态、软删除标记等条件的搜索。
  - 列表展示各开关状态，支持新增/编辑/删除/批量删除与 Excel 导出。
  - 新增“隐私设置统计”卡片，展示：
    - 总用户数
    - 已同意隐私协议数
    - 已开启匿名授权数
    - 已开启自动清理数
    - 已开启本地存储数
  - 统计接口复用 `/TUserPrivacySetting/List`，前端自行在内存中聚合。

- `DataExportRequestList.vue`
  - 支持按用户、状态、时间范围、软删除标记等条件查询导出申请。
  - 列表展示数据类型、导出格式、状态、文件大小、创建时间等。
  - 提供导出 Excel 功能，便于运营或合规审计。

- `DataDeleteRequestList.vue`
  - 支持按用户、状态、时间范围、软删除标记等条件查询删除申请。
  - 列表展示数据类型、删除原因、状态、删除记录数等。
  - 同样支持导出 Excel。

## 五、小程序功能调试与测试指南（重点）

### 5.1 基础准备

1. **确认数据库已执行隐私模块相关脚本**：
   - `隐私设置与数据权利模块-数据库修改.sql`
   - 保证 `tdataexportrequest`、`tdatadeleterequest` 存在，`tuserprivacysetting` 中新增 `DataRetentionEnabled/DataRetentionMonths` 字段已生效。
2. **确认后端 Spring Boot 已启动**，并且 `HealthControl.uniapp/utils/config.js` 中的 `API_BASE_URL` 指向正确。
3. **保证登录流程可用**，可以在小程序内正常获取 `Token` 与 `UserId`。

### 5.2 测试隐私设置主页面

1. 登录小程序，从 `UserCenter` 进入“隐私设置”。
2. 检查初始状态：
   - 若用户首次进入，应看到所有开关关闭，协议为“未同意”。
3. 测试本地存储开关：
   - 打开开关后，调用 `/Front/Privacy/UpdateSetting` 应返回成功，列表/详情页可根据需要决定是否缓存数据（当前实现主要为配置占位）。
   - 关闭开关应弹出确认弹窗，确认后再提交。
4. 测试匿名授权总开关：
   - 打开时应弹出说明弹窗，确认后 `DataAnonymousAuth` 为 `true`。
   - 关闭时也有说明弹窗，强调不会影响已授权的历史记录。
5. 测试数据留存与自动清理：
   - 首次开启时，选择一个保留时长（默认 6 个月），弹窗确认后后端应保存 `DataRetentionEnabled=true` 且 `DataRetentionMonths=6`。
   - 调整保留时长应立即保存设置。
   - 关闭自动清理时同样需确认弹窗。
6. 测试隐私协议：
   - 在隐私设置页或协议详情页点击“我已阅读并同意”，应看到状态变为“已同意”，后端 `PrivacyAgreeStatus=true`。

### 5.3 测试数据导出申请与下载

1. 在隐私设置页点击“申请导出数据”，进入 `DataExportRequest`。
2. 选择若干数据类型（至少一项），格式保持 `JSON`。
3. 填写可选说明，点击“提交导出申请”：
   - 确认前端提示“提交成功”，并能在返回结果中拿到有效 `RequestId`。
4. 在弹出的确认框中选择“立即下载”：
   - 前端通过 `uni.downloadFile` + `uni.saveFile` 下载一个 JSON 文件。
   - 打开该文件，应包含用户信息（已脱敏）及对应数据集合字段。
5. 返回“数据权利申请记录”页：
   - 该导出申请应出现在“导出申请”中，状态为 `Completed`。

### 5.4 测试数据删除申请与异步处理

1. 在隐私设置页点击“申请删除数据”，进入 `DataDeleteRequest`。
2. 勾选若干数据类型，并选择删除原因（可选）。
3. 勾选确认复选框，否则前端应提示“请先勾选确认”。
4. 提交删除申请：
   - 前端弹窗确认后调用 `/Front/Privacy/DeleteRequest`，成功后弹出“已提交”并跳转到申请记录页。
5. 在“数据权利申请记录”页中，切换到“删除申请”标签：
   - 刚提交的记录应显示为 `Pending`。
6. 等待 1–2 分钟（或在开发环境缩短 cron 表达式间隔）：
   - 定时任务 `PrivacyDeleteRequestJob` 会将状态更新为 `Processing` → `Completed` 或 `Failed`。
   - 再次下拉刷新申请记录页，应看到状态更新为 `Completed`，`DeleteCount` 显示实际软删除的记录数。

### 5.5 测试数据权利申请记录分页

1. 通过多次导出/删除请求，制造多条导出/删除申请记录。
2. 在 `DataRightsHistory` 页中：
   - 使用下拉刷新，确认会从第一页重新加载数据。
   - 上拉到底部，应触发加载下一页数据（如果总数超过一页大小）。
   - 重复上拉，直到 `exportHasMore` 和 `deleteHasMore` 都变为 `false`，此时不再触发加载。

### 5.6 测试模型授权明细与标注联动

1. 在小程序中通过“确诊标注”相关页面创建若干标注记录，并在其中开启“用于模型优化”的授权开关。
2. 打开 `AuthUsageDetail` 页：
   - 默认展示已授权记录列表，可按照时间范围与检测结果进行筛选。
3. 点击某一条“取消授权”：
   - 前端弹出风险提醒弹窗，确认后调用 `/Front/ModelLabel/UpdateAuth`。
   - 重新加载列表，该条记录应变为“未授权”。
4. 返回模型标注列表页，确认授权状态显示同步更新。

## 七、常见问题与排查

### 7.1 小程序提示“用户未登录”

- 确认前端是否在调用隐私相关接口前完成登录，并在请求头中携带了正确的 `Authorization`。
- 检查 `BaseContext` 的登录拦截是否对 `/Front/Privacy/...` 和 `/Front/ModelLabel/...` 生效。

### 7.2 下载导出文件失败

- 确认后端是否正确拼接 `Content-Type` 与 `Content-Disposition` 响应头。
- 检查小程序端 `API_BASE_URL` 是否指向了可访问的服务器地址。
- 若返回 401/403，通常是 Token 过期或权限校验未通过。

### 7.3 删除申请一直停留在 Pending 状态

- 检查 `PrivacyDeleteRequestJob` 是否被正确注册：
  - Spring Boot 是否开启了 `@EnableScheduling`。
  - 应用日志中是否有“处理删除申请失败”的错误信息。
- 检查 `tdatadeleterequest` 的 `IsDelete` 标记是否被误置为 `true`，导致定时任务筛选不到记录。

### 7.4 自动清理未生效

- 确认对应用户的 `DataRetentionEnabled=true` 且 `DataRetentionMonths>0`。
- 检查需要清理的数据表中，`CreationTime` 或等价字段是否早于阈值时间。
- 检查 `PrivacyDataRetentionJob` 是否正常执行，可通过日志中“自动清理用户历史数据：UserId=..., Months=..., Affected=...”来确认。

## 八、后续可优化点

1. **导出任务真正异步化**  
   将当前“即时导出”模式改为真正的异步任务（如通过队列或独立 Job 生成文件并落盘），`tdataexportrequest.FileUrl` 存储实际文件路径，前端根据任务状态展示“准备中/可下载”。

2. **更细粒度的授权管理**  
   支持按数据类型或业务场景进行更细分的授权（例如单独控制音频数据、问卷数据、日志数据是否可用于算法优化）。

3. **隐私协议版本管理**  
   在 `tuserprivacysetting` 中引入 `ProtocolVersion` 字段，配合协议内容版本号，支持在协议升级时强制用户重新确认。

4. **数据权利处理 SLA 监控**  
   在后台增加数据权利处理看板，如导出/删除平均处理时长、超期任务告警等，辅助合规运营。
